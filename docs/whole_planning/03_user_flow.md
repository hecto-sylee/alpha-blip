# 사용자 핵심 플로우

> 시스템 구분(S1~S5, 공통)은 [02_feature_spec.md](./02_feature_spec.md) 참고.

---

## 0. 온보딩 플로우 (공통)

```
회원가입
  → 보호자 기본 정보 입력
  → 반려동물 프로필 등록 (F-02)
  → [강아지] 산책 스타일 설정 / 위치 권한 허용
  → 홈 진입
```

---

## 1. 산책 매칭 플로우 (S1)

매칭 성사 후 산책 종료 시 곧바로 S3 기록 시스템으로 이어진다.

```
산책 시작 (F-01 지도 활성화)
  → 지도에서 근처 강아지 확인
  → 프로필 확인 (F-02)
  → [같이 산책하기] 요청 (F-03)
  → 상대방 수락
  → 매칭 세션 시작 (F-04)
  → 함께 산책
  → 산책 종료
  → 매칭 로그 자동 저장 (F-05)
  → ▶ 기록 남기기 화면 자동 진입 (S3 / F-10)
```

---

## 2. 기록 플로우 (S3)

매칭 유무와 무관하게 산책 종료 후 항상 진입한다. F-10이 콘텐츠 생산 레이어이며, 공개 범위에 따라 F-11(방)·S5(동네)로 분기된다.

### 2-1. 산책 기록 작성 (F-10)

```
산책 종료 (매칭 산책 또는 혼자 산책)
  → 기록 남기기 화면 진입
  → 영상/사진 업로드 또는 촬영 (선택)
  → 텍스트·스티커·그림 꾸미기 (선택)
  → 공개 범위 설정
       ├─ 일기 (나만 보기)
       ├─ 방(Room) 공유 (F-11)
       └─ 동네 공유 (S5 / F-13)
  → 저장
  → (매칭 산책인 경우) 친구 요청 제안 (S2 / F-06)
```

### 2-2. 방(Room) 공유 (F-11)

```
방 생성 (반려동물 조합 또는 그룹 이름 지정)
  → 멤버 초대
       ├─ 매칭 이력 1회 이상인 상대 초대
       ├─ 친구 초대 (S2 / F-06)
       └─ 방 코드 공유 (앱 외부 공유 가능)
  → F-10 기록을 방에 업로드
  → 방 히스토리 확인
  → (새 기록 업로드 시) 참여자 알림
```

## 3. 소셜/관계 플로우 (S2)

S1 매칭 이력이 있어야 진입 가능하다.

```
매칭 이력 확인 (F-05)
  → 친구 요청 / 수락 (F-06)
  → 반복 산책으로 친밀도 누적 (F-07)
  → 산책 호출 단계 상승 (F-08)
       ├─ 1단계: "같이 산책할래요?" 알림
       ├─ 2단계: 상대 산책 시작 알림 수신 (양측 동의)
       └─ 3단계: 정기 산책 제안 / 빠른 호출 / 그룹 산책 초대
```

---

## 4. Pet Space 플로우 (S4)

별도로 만들어가는 공간이 아니라, S1 매칭과 S3 기록이 쌓이며 자동으로 채워진다.

```
내 Pet Space 진입
  → 산책 기록·매칭 이력 피드 확인 (S1·S3 데이터 자동 반영)
  → 공간 꾸미기 (배경, 소품, 스티커)
  → 친구 반려동물의 Pet Space 방문 (S2 친구 관계)
  → 방문 반응 남기기
```

---

## 5. 커뮤니티 플로우 (S5)

F-10 기록이 콘텐츠 공급원이 된다.

```
동네 피드 진입 (F-13)
  → 같은 동네 반려동물 기록 피드 탐색
  → 게시글 확인 / 반응
  → F-10 기록을 동네 공개로 피드에 공유
  → 동네 반려인 발견
       ├─ S1 매칭 요청
       └─ S2 친구 요청
  → (이후 확장) 종별 니치 커뮤니티 진입 (F-14)
```
