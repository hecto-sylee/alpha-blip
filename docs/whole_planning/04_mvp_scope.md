## Phase별 기능 확장

| 단계 | 목표 | 핵심 기능 |
|---|---|---|
| **Phase 1. 웹 MVP** | 핵심 UX 검증 | 위치 권한, 산책 시작/종료, 근처 강아지 지도 표시, 프로필 확인, 산책 요청/수락, 매칭 로그 |
| **Phase 2. 앱 MVP** | 실제 산책 상황 검증 + 기록 루틴 형성 | 앱 기반 위치 수집, 푸시 알림, 친구 요청, 반복 매칭 기록, 산책 후 피드백, 기본 친밀도, 산책 기록 다이어리 |
| **Phase 3. 정식 서비스** | 위치 기반 P2P 산책 네트워크 + 반려동물 중심 공간 구축 | 친밀도 시스템, 산책 호출, 그룹 산책, 방(Room) 기반 공유 기록, Pet Space, 동네 커뮤니티, AI 추천 |
| **Phase 4. 플랫폼 확장** | 모든 반려동물로 확장 | 니치 커뮤니티 (종별), 강아지 외 반려동물 활동 기록 레이어 확대, 커뮤니티 종별 분화 |

---

## Phase별 레이어 확장 구조

```
Phase 1~2:
  [레이어 1] 산책 매칭  →  강아지 전용, 핵심 검증

Phase 2~3:
  [레이어 1] 산책 매칭
  [레이어 2] 활동 기록 / 다이어리  →  강아지부터 시작, 이후 모든 반려동물

Phase 3~4:
  [레이어 1] 산책 매칭
  [레이어 2] 활동 기록 / 다이어리
  [레이어 3] 커뮤니티  →  동네 커뮤니티 → 종별 니치 커뮤니티로 분화
```
