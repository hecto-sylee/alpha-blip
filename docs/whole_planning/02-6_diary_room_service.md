# 02-6. 산책 기록 다이어리 & 방(Room) 서비스

> 관련 기능: F-10 (산책 기록 다이어리), F-11 (방 기반 공유 기록)
> MVP 포함: F-10 Phase 2 / F-11 Phase 3

---

## 서비스 개요

산책 완료 후 기록을 남기는 개인 다이어리와, 친구들과 기록을 공유하는 방(Room) 구조를 제공하는 서비스.
산책 기록이 쌓이는 경험을 통해 루틴 형성과 앱 재방문을 유도한다.
개인 다이어리(F-10)가 콘텐츠 생산 레이어가 되고, 방(F-11)은 그 위에 올라오는 공유 레이어로 구성된다.

```
개인 다이어리 (F-10)  →  방(Room) 공유 (F-11)  →  동네 커뮤니티 공유 (F-13, 선택)
```

---

## F-10. 산책 기록 다이어리

**목적:** 산책 완료 후 기록을 남겨 루틴 형성과 앱 재방문을 유도한다.

> 매칭 유무와 무관하게 산책 완료 기록은 항상 남길 수 있다.
> 매칭이 있었다면 상대방 반려견 정보와 함께 기록된다.

### 기록 진입 플로우

```
산책 종료 버튼 클릭
  → 기록 남기기 화면 자동 진입
  → 사진/영상 업로드 (선택)
  → 텍스트·스티커·그림 등으로 꾸미기 (선택)
  → 공개 범위 설정 (일기 / 방 공유 / 동네 공유)
  → 저장
```

### 기록 포함 정보

| 항목 | 저장 방식 |
|---|---|
| 산책 날짜 / 시간 / 거리 | 자동 |
| 함께한 반려견 (매칭이 있었을 경우) | 자동 |
| 사진 / 영상 | 사용자 업로드 |
| 텍스트, 스티커, 그림 | 사용자 입력 |
| 공개 범위 | 사용자 선택 |

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 캘린더 뷰 | 날짜별 산책 기록 확인 |
| 기록 꾸미기 에디터 | 사진/영상 위에 텍스트·스티커 추가 |
| 산책 스탯 | 누적 거리, 산책 횟수, 연속 일수 |
| 공개 범위 선택기 | 일기 / 방 공유 / 동네 공유 |

### 기획 원칙

- 미디어 없이 텍스트만으로도 기록 가능하도록 설계
- 다꾸(다이어리 꾸미기) 감성 + 운동 기록 앱 스탯 구조
- 기록을 남기는 행위 자체가 산책 루틴의 마무리가 되도록 UX 설계

## F-11. 방(Room) 기반 공유 기록

**목적:** 특정 반려견 조합 또는 친구 그룹 단위로 기록을 모아 공유하는 공동 공간을 만든다.

### 방 개요

- 방은 특정 반려견 조합 또는 친구 그룹 단위로 생성
- 방 안에서 매칭 기록, 산책 완료 기록 등을 함께 쌓음
- 개인 다이어리(F-10)에서 꾸민 기록을 방에 업로드하는 방식으로 연결

### 방 참여 방법

| 방법 | 설명 |
|---|---|
| 매칭 이력 기반 초대 | 1회 이상 매칭 기록이 있는 상대를 초대 |
| 친구 초대 | 이미 친구 추가된 상대를 초대 |
| 방 코드 공유 | 방 코드를 직접 공유해 참여 (앱 외부 공유 가능) |

### 주요 기능

| 기능 | 설명 |
|---|---|
| 방 생성 | 누구나 생성 가능, 반려견 조합 또는 그룹 이름 지정 |
| 멤버 초대 | 매칭 이력 / 친구 / 방 코드 세 가지 방식 |
| 기록 업로드 | 개인 다이어리 기록을 방에 공유 |
| 방 히스토리 | 함께한 산책 기록 아카이브 타임라인 |
| 방 참여 관리 | 초대 / 나가기 |
| 방 알림 | 새 기록 업로드 시 참여자 알림 |

### 활용 시나리오

- 자주 같이 산책하는 친구 반려견과의 기록 모아보기
- 그룹 산책 히스토리 아카이브
- 반려견 조합별 추억 공간
