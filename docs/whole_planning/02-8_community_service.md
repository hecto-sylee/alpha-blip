# 02-8. 커뮤니티 서비스

> 관련 기능: F-13 (동네 커뮤니티), F-14 (니치 커뮤니티)
> MVP 포함: Phase 3 (동네 커뮤니티) / Phase 4 (니치 커뮤니티)

---

## 서비스 개요

동네 기반 커뮤니티와 반려동물 종별 니치 커뮤니티를 제공하는 서비스.
기존 커뮤니티(네이버 카페, 당근마켓 등)의 페인 포인트를 공략하며,
이 앱의 핵심 차별화 포인트인 "내 동물을 아는 커뮤니티"를 구현한다.

### 기존 커뮤니티 vs 이 서비스

| 기존 서비스 페인 포인트 | 이 서비스의 대안 |
|---|---|
| 글마다 내 동물 반복 소개 | 프로필 기반 — 내 동물을 이미 아는 상태 |
| 지역성 약함 (전국 단위 카페) | 동네 기반 커뮤니티 |
| 온라인→오프라인 연결 없음 | 동네 매칭·모임 연결 |
| 텍스트 중심, 낡은 UX | 영상/다이어리 기반 기록 UX |
| 기록과 커뮤니티 분리 | 기록 → 커뮤니티 공유가 한 흐름 |
| 신뢰 기반 관계 없음 | 실제 산책 기록이 있는 사용자 기반 관계 |

---

## F-13. 동네 커뮤니티

**목적:** 동네 기반으로 산책·매칭 기록을 다른 사용자들과 공유하고, 지역 반려인 네트워크를 형성한다.

### 주요 기능

| 기능 | 설명 |
|---|---|
| 동네 피드 | 같은 동네 반려동물 산책 기록을 피드 형태로 확인 |
| 동네 반려동물 발견 | 자주 산책하는 근처 반려동물·반려인 탐색 |
| 게시글 작성 | 동네 공개 기록 또는 자유 게시글 |
| 댓글 / 반응 | 커뮤니티 상호작용 |
| 공개 범위 설정 | 전체 공개 / 동네만 / 비공개 |
| 모임 제안 | 동네 반려인 그룹 산책 또는 모임 제안 |

### 기획 원칙

- 동네 기반 게시글은 정확한 위치를 노출하지 않는다
- 개인 다이어리(F-10) 기록을 동네 피드로 공유하는 흐름으로 자연스럽게 연결
- 콘텐츠 생산 부족 초반에는 산책 기록 자동 공유 옵션(선택)으로 피드 채움
- 기존 산책 매칭 이력이 있는 사용자 간 신뢰 기반 커뮤니티로 차별화

### 피드 콘텐츠 유형

| 유형 | 설명 |
|---|---|
| 산책 기록 공유 | 개인 다이어리에서 동네 공개로 설정한 기록 |
| 매칭 완료 기록 | 매칭 후 공유 동의 시 피드에 노출 |
| 자유 게시글 | 반려동물 관련 자유 글 |
| 모임 제안 | 그룹 산책 또는 모임 공지 |

## F-14. 니치 커뮤니티 (종별 커뮤니티)

**목적:** 반려동물 종별 관심사 기반 커뮤니티를 제공한다.

### 종별 커뮤니티 구성

| 반려동물 | 커뮤니티 주요 콘텐츠 | 산책 매칭 확장 가능성 |
|---|---|---|
| 고양이 | 품종별 건강 정보, 실내 환경 구성 | 하네스 산책 유저 한정 가능 |
| 토끼 | 식이/질병 정보, 운동공간 공유 | 야외 놀이 모임 (매우 니치) |
| 앵무새/조류 | 훈련·언어 발달, 품종별 특성 | 공원 외출 모임 가능성 있음 |
| 파충류 | 사육 환경, 먹이 정보 | 매칭 없음, 기록 레이어만 유효 |
| 소동물 | 수명·질병, 케어 정보 | 매칭 없음, 기록 레이어만 유효 |

### 확장 원칙

- 강아지 커뮤니티(동네 커뮤니티, F-13) 활성화 이후 단계적으로 확장
- 프로필에 등록된 반려동물 종 기반으로 자동 커뮤니티 추천
- 기록 레이어(F-10)가 니치 커뮤니티의 콘텐츠 생산 기반
