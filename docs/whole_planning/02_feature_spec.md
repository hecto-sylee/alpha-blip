## 기록 레이어 구조 (F-10 / F-11 / F-13)

산책 기록 관련 기능은 독립적으로 작동하는 것이 아니라,
**F-10이 콘텐츠 생산 레이어, F-11과 F-13이 그 위에 올라오는 공유 레이어**로 구성된 계층 구조다.

```
F-10 산책 기록 다이어리 (콘텐츠 생산 레이어)
  └─ 공개 범위 설정
       ├─ 일기 (나만 보기)
       ├─ F-11 방(Room) 공유  → 특정 친구와 공유 (닫힌 공유)
       └─ F-13 동네 커뮤니티  → 동네 반려인과 공유 (열린 공유)
```

F-10에서 기록을 만들지 않으면 F-11과 F-13 모두 콘텐츠가 없다.
F-11, F-13은 F-10에서 꾸며진 기록을 **어디에 공유할지 결정**하는 레이어다.

### 루틴 형성 메커니즘

```
앱 실행 → 산책 시작 → [매칭 있음/없음] → 산책 완료 → 기록 남기기(F-10) → 저장
         ↑__________________________________________________|
```

이 루프가 반복될수록 "산책 = 앱을 켜는 것"이 루틴으로 자리잡는다.
매칭이 없어도 기록을 남길 수 있다는 점이 핵심 — 앱 사용이 매칭 성사 여부에 종속되지 않는다.

## F-01. 실시간 산책 지도

**목적:** 지금 근처에 함께 산책 가능한 반려견이 있는지 확인한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 산책 시작 버튼 | 위치 공유 시작, 지도 활성화 |
| 현재 위치 표시 | 내 위치 마커 |
| 근처 강아지 아이콘 | 산책 중인 다른 사용자 표시 |
| 거리 표시 | 아이콘 옆 대략적 거리 (예: 200m) |
| 기본 프로필 미리보기 | 아이콘 탭 시 이름·견종·성격 태그 요약 |
| 필터 | 크기·성격 등으로 필터링 |

### 지도 표시 정보

강아지 이름 / 견종 / 크기 / 나이 / 성격 태그 / 현재 거리 / 산책 상태

### 기획 원칙

- 정확한 좌표를 그대로 노출하지 않고, 대략적 위치 또는 일정 반경 내 아이콘으로 표시한다
- 산책 종료 시 즉시 지도에서 사라진다

## F-02. 반려견 프로필

**목적:** 같이 산책하기 전 반려견의 성향과 기본 정보를 확인한다.

### 프로필 항목

| 항목 | 필수 여부 |
|---|---|
| 이름 | 필수 |
| 사진 | 필수 |
| 견종 | 필수 |
| 나이 | 필수 |
| 성별 | 필수 |
| 크기 | 필수 |
| 중성화 여부 | 선택 |
| 성격 태그 | 필수 (최소 1개) |
| 사회성 | 선택 |
| 활동량 | 선택 |
| 산책 스타일 | 선택 |
| 선호 친구 유형 | 선택 |
| 주의사항 | 선택 |

### 성격 태그 목록 (예시)

`사람 좋아함` `강아지 좋아함` `소심함` `활발함` `천천히 걷기 선호` `뛰는 산책 선호` `대형견 무서워함` `짖음 있음`

## F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 반려견과 즉시 산책 연결을 시도한다.

### 사용자 플로우

```
프로필 확인
  → [같이 산책하기] 버튼 클릭
  → 상대방에게 요청 알림 발송
  → 상대방 수락 또는 거절
  → 수락 시 매칭 성사 (F-04 시작)
```

### 요청 상태

| 상태 | 설명 |
|---|---|
| 요청 대기 | 상대방 응답 전 |
| 수락 | 매칭 세션 생성 |
| 거절 | 요청 종료, 상대방에게 노출 최소화 |
| 시간 만료 | N분 내 응답 없으면 자동 만료 (시간 TBD) |
| 취소 | 요청자가 직접 취소 |

### 기획 원칙

- 요청은 가볍고 부담이 적어야 한다
- 거절은 상대방에게 부정적으로 노출되지 않도록 설계한다

## F-04. 매칭 세션

**목적:** 함께 산책하는 동안의 상태를 관리하고, 종료 후 로그를 생성한다.

### 세션 정보

| 항목 | 설명 |
|---|---|
| 참여 사용자 | 요청자 + 수락자 |
| 참여 반려견 | 양측 반려견 정보 |
| 시작 시간 | 수락 시점 |
| 종료 시간 | 산책 종료 버튼 또는 자동 종료 |
| 산책 위치 범위 | 대략적 산책 구역 |
| 동행 시간 | 세션 지속 시간 |
| 동행 거리 | 이동 거리 (선택) |
| 산책 상태 | 진행 중 / 종료 |

### 세션 중 제공 기능

| 기능 | 설명 |
|---|---|
| 합류 안내 | 만남 장소 안내 |
| 간단 메시지 | 짧은 텍스트 또는 이모지 |
| 산책 종료 | 세션 종료 → 로그 생성 → 기록 화면(F-10) 진입 |
| 신고 / 차단 | 부적절한 사용자 신고 |
| 매칭 취소 | 세션 중 이탈 |

---

## F-05. 매칭 로그

**목적:** 일회성 만남을 데이터로 축적하고, 반복 만남과 관계 형성의 기반을 만든다.

### 로그 항목

| 항목 | 저장 방식 |
|---|---|
| 함께 산책한 강아지 | 자동 |
| 산책 날짜 | 자동 |
| 산책 시간 | 자동 |
| 산책 거리 | 자동 (선택) |
| 만남 횟수 | 누적 카운트 |
| 사진 | 사용자 업로드 (선택) |
| 산책 후 피드백 | 사용자 입력 (선택) |

### 로그 활용처

친구 추천 / 친밀도 계산 / 산책 히스토리 / 반복 매칭 유도 / 다이어리 기본 데이터(F-10)

---

## F-06. 친구 시스템

**목적:** 우연한 산책 만남을 지속적인 관계로 전환한다.

**활성화 조건:** 한 번 이상 매칭이 성공한 사용자에게만 친구 요청 가능

### 주요 기능

| 기능 | 설명 |
|---|---|
| 친구 요청 / 수락 / 거절 | 기본 친구 관계 관리 |
| 친구 목록 | 친구 목록 + 마지막 산책 정보 |
| 친구별 산책 기록 | 함께한 산책 히스토리 |
| 친구 알림 설정 | 알림 ON/OFF 개별 설정 |
| 친구 차단 / 삭제 | 관계 종료 |

### 기획 원칙

아무에게나 친구 요청을 보내는 구조가 아니라, 실제로 한 번 이상 산책한 상대를 중심으로 관계를 형성한다.

---

## F-07. 친밀도 시스템

**목적:** 반복 만남을 유도하고, 강아지 간 관계 형성을 서비스의 핵심 재미 요소로 만든다.

> 명칭 후보: 친밀도 / 산책 케미 / 친구 레벨 / 펫 케미 지수

### 점수 변동 기준

| 상승 요인 | 하락/정체 요인 |
|---|---|
| 같은 강아지와 반복 산책 | 장기간 미만남 |
| 친구 요청 수락 | 반복적인 요청 거절 |
| 산책 후 긍정 피드백 | 산책 후 부정 피드백 |
| 일정 시간 이상 함께 산책 | 친구 삭제 |
| 호출 후 실제 산책 성사 | 차단 |
| 방(Room)에 공유 기록 추가 | — |

### 기획 원칙

- 하락은 사용자에게 직접적으로 강조하지 않는다
- 부정 피드백은 상대방에게 공개하지 않는다

---

## F-08. 산책 호출 기능

**목적:** 처음 만난 상대와 다시 만날 수 있는 구조를 만든다.

### 단계별 기능

| 단계 | 조건 | 가능한 기능 |
|---|---|---|
| **1단계: 산책 친구** | 1회 매칭 또는 친구 수락 | "같이 산책할래요?" 알림 전송 / 과거 산책 로그 확인 |
| **2단계: 자주 만나는 친구** | 반복 산책 + 긍정 피드백 + 상호 친구 관계 | 상대 산책 시작 시 알림 수신 (양측 동의 필요) / 대략적 산책 구역 확인 |
| **3단계: 베스트 산책 메이트** | 높은 친밀도 + 반복 산책 + 상호 동의 | 정기 산책 제안 / 빠른 호출 / 그룹 산책 초대 / AI 기반 산책 타이밍 추천 |

## F-09. 개인정보 보호 설정

**목적:** 사용자가 언제, 누구에게, 어느 수준까지 위치를 공유하는지 직접 통제한다.

### 주요 설정 항목

| 설정 | 설명 |
|---|---|
| 위치 공유 ON/OFF | 산책 중에만 위치 공유 |
| 산책 종료 시 자동 비공개 | 종료 즉시 지도에서 제거 |
| 대략적 위치 표시 | 정확한 좌표 대신 반경 내 표시 |
| 친구별 알림 허용 설정 | 특정 친구 알림만 수신 |
| 특정 시간대 알림 제한 | 야간 등 알림 차단 시간 설정 |
| 차단 / 신고 | 특정 사용자 차단 |
| 마지막 위치 자동 만료 | 일정 시간 후 위치 데이터 삭제 |
| 집 주변 위치 노출 방지 | 홈 반경 내 위치 비공개 처리 |
| 기록 공개 범위 설정 | 다이어리/방(Room) 공개 범위 제어 |
| 동네 커뮤니티 활동 공개 설정 | 동네 피드 활동 공개/비공개 |

## F-10. 산책 기록 다이어리

**목적:** 산책 완료 후 기록을 남겨 루틴 형성과 앱 재방문을 유도하며, F-11(방 공유)·F-13(동네 커뮤니티)의 콘텐츠 생산 레이어 역할을 한다.

> 매칭 유무와 무관하게 산책 완료 기록은 항상 남길 수 있다.
> 매칭이 있었다면 상대방 반려견 정보와 함께 기록된다.

### 기록 기본 포맷

기본 포맷은 **영상** (산책 중 촬영하거나, 사진들로 자동 구성).
영상/사진 위에 텍스트, 스티커, 그림을 얹어 꾸밀 수 있다 (다꾸 감성).
미디어 없이 텍스트만으로도 기록 가능하다.

### 기록 진입 플로우

```
산책 종료 버튼 클릭
  → 기록 남기기 화면 자동 진입
  → 영상/사진 업로드 또는 촬영 (선택)
  → 텍스트·스티커·그림 등으로 꾸미기 (선택)
  → 공개 범위 설정
       ├─ 일기 (나만 보기)
       ├─ 방(Room) 공유 (F-11)
       └─ 동네 공유 (F-13)
  → 저장
```

### 기록 포함 정보

| 항목 | 저장 방식 |
|---|---|
| 산책 날짜 / 시간 / 거리 | 자동 |
| 함께한 반려견 (매칭이 있었을 경우) | 자동 |
| 영상 / 사진 | 사용자 업로드 또는 촬영 |
| 텍스트, 스티커, 그림 | 사용자 입력 |
| 공개 범위 | 사용자 선택 |

### 주요 UI

| 요소 | 설명 |
|---|---|
| 캘린더 뷰 | 날짜별 산책 기록 확인 |
| 기록 꾸미기 에디터 | 영상/사진 위에 텍스트·스티커 추가 |
| 산책 스탯 | 누적 거리, 산책 횟수, 연속 일수 |
| 공개 범위 선택기 | 일기 / 방 공유 / 동네 공유 |

### 기획 원칙

- 기록을 남기는 행위가 산책 루틴의 마무리가 되도록 UX 설계 (열린 루프 닫기)
- 다꾸(다이어리 꾸미기) 감성 + 운동 기록 앱 스탯 구조
- 미디어 없이 텍스트만으로도 기록 가능 (진입 장벽 최소화)

## F-11. 방(Room) 기반 공유 기록

**목적:** F-10에서 생산된 기록을 특정 반려견 조합 또는 친구 그룹 단위로 모아 공유하는 **닫힌 공유 레이어**.

> F-10 없이는 방(Room)에 올릴 콘텐츠가 없다. F-11은 F-10의 공유 목적지다.

### 방 개요

- 방은 특정 반려견 조합 또는 친구 그룹 단위로 생성
- 방 안에서 매칭 기록, 산책 완료 기록 등을 함께 쌓음
- F-10에서 꾸며둔 개인 기록을 방에 업로드하는 방식으로 연결

### 주요 기능

| 기능 | 설명 |
|---|---|
| 방 생성 | 친구 관계 기반, 반려견 조합 지정 |
| 기록 업로드 | 개인 다이어리 기록을 방에 공유 |
| 방 히스토리 | 함께한 산책 기록 아카이브 |
| 방 참여 관리 | 초대 / 나가기 |

### 방 개설 조건

한 번 이상 매칭 성공 + 친구 관계인 상대와만 방 생성 가능

### 활용 시나리오

- 자주 같이 산책하는 친구 반려견과의 기록 모아보기
- 그룹 산책 히스토리 아카이브

## F-12. Pet Space

**목적:** 반려동물이 자신의 공간을 가지고, 기록과 관계가 반려동물 중심으로 쌓이도록 한다.

### Pet Space 구성

| 구성 요소 | 설명 |
|---|---|
| 피드 | 산책 기록, 매칭 기록이 반려동물 관점에서 쌓임 |
| 공간 꾸미기 | 주인이 반려동물의 방/집을 꾸며줌 (배경, 소품, 스티커 등) |
| 방문 | 매칭으로 친해진 다른 반려동물의 공간 방문 |
| 반려동물 소개 | 의인화 톤으로 작성된 반려동물의 자기소개 |

### 의인화 UX

```
일반적인 반려동물 앱: "보미, 말티즈, 3살"
의인화 UX:            "안녕, 나는 보미야. 느린 산책이 제일 좋아."
```

의인화는 기능이 아니라 서비스 전반의 톤앤매너로 자연스럽게 스며든다.

### 실세계-디지털 연결

```
실세계 레이어:  산책 매칭 → 실제로 만남
디지털 레이어:  Pet Space → 친구 집에 디지털로 놀러감
```

## F-13. 동네 커뮤니티

**목적:** F-10에서 생산된 기록을 동네 반려인과 공유하는 **열린 공유 레이어**. 지역 반려인 네트워크를 형성한다.

> F-10 없이는 피드를 채울 콘텐츠가 없다. F-13은 F-10의 또 다른 공유 목적지다.

### 주요 기능

| 기능 | 설명 |
|---|---|
| 동네 피드 | 같은 동네 반려동물 산책 기록을 피드 형태로 확인 |
| 동네 반려동물 발견 | 자주 산책하는 근처 반려동물·반려인 탐색 |
| 게시글 작성 | 동네 공개 기록 또는 자유 게시글 |
| 댓글 / 반응 | 커뮤니티 상호작용 |
| 공개 범위 설정 | 전체 공개 / 동네만 / 비공개 |

### 기획 원칙

- 동네 기반 게시글은 정확한 위치를 노출하지 않는다
- 개인 다이어리(F-10) 기록을 동네 피드로 공유하는 흐름으로 연결
- 콘텐츠 생산 부족 초반에는 산책 기록 자동 공유 옵션으로 피드 채움

## F-14. 니치 커뮤니티 (종별 커뮤니티)

**목적:** 반려동물 종별 관심사 기반 커뮤니티를 제공한다.

### 종별 커뮤니티 구성

| 반려동물 | 커뮤니티 주요 콘텐츠 |
|---|---|
| 고양이 | 품종별 건강 정보, 실내 환경 구성 |
| 토끼 | 식이/질병 정보, 운동공간 공유 |
| 앵무새/조류 | 훈련·언어 발달, 품종별 특성 |
| 파충류 | 사육 환경, 먹이 정보 |
| 소동물 | 수명·질병, 케어 정보 |

### 기획 원칙

- 강아지 커뮤니티(동네 커뮤니티) 활성화 이후 단계적으로 확장
- 프로필 기반으로 유저의 반려동물 종이 이미 등록된 상태에서 자동 추천
- 기록 레이어(F-10)가 니치 커뮤니티의 콘텐츠 생산 기반이 됨
