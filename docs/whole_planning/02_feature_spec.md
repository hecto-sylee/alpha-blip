## 3. 레이어 구조 및 시스템-기능 매핑

### 전체 레이어 구조

```
[공통]  F-02 반려동물 프로필 / F-09 개인정보 보호 설정
         ↓ (모든 시스템에 기반 제공)

[S1]  산책 매칭 ── F-01 지도 / F-03 요청 / F-04 세션 / F-05 로그
         ↓ (매칭 이력 → S2 소셜 진입 조건 / S3 기록 기본 데이터)

[S2]  소셜/관계 ── F-06 친구 / F-07 친밀도 / F-08 호출
         ↓ (친구 관계 → S3 방(Room) 개설 조건)

[S3]  기록 ── F-10 다이어리 (콘텐츠 생산)
               └─ F-11 방(Room) 공유 (닫힌 공유, S2 친구와)
               └─ F-13 동네 커뮤니티 공유 (열린 공유, S5로 연결)

[S4]  Pet Space ── F-12  (F-02 프로필 + S1/S3 기록이 채워주는 공간)

[S5]  커뮤니티 ── F-13 동네 커뮤니티 / F-14 니치 커뮤니티
                  (F-10 기록이 콘텐츠 공급원)
```

### 공통 기능이 각 시스템에서 쓰이는 방식

| 공통 기능 | S1 산책 매칭 | S2 소셜/관계 | S3 기록 | S4 Pet Space | S5 커뮤니티 |
|---|---|---|---|---|---|
| F-02 반려동물 프로필 | 지도 아이콘·매칭 필터 기반 | 친밀도 계산 기반 | 기록에 반려동물 정보 포함 | Pet Space의 정체성 | 커뮤니티 글쓴이 맥락 |
| F-09 개인정보 보호 설정 | 위치 공개 범위 | 친구별 알림 설정 | 기록 공개 범위 | Pet Space 공개 여부 | 동네 피드 공개 설정 |

### S3 기록 시스템 내 콘텐츠 레이어 구조

```
F-10 산책 기록 다이어리 (콘텐츠 생산 레이어)
  └─ 공개 범위 설정
       ├─ 일기 (나만 보기)
       ├─ F-11 방(Room) 공유  →  특정 친구와 공유 (닫힌 공유)
       └─ F-13 동네 공유      →  동네 반려인과 공유 (열린 공유, S5)
```

F-10이 없으면 F-11과 F-13 모두 콘텐츠가 없다.
F-11, F-13은 F-10에서 만들어진 기록을 **어디에 공유할지** 결정하는 레이어다.

### 루틴 형성 메커니즘 (S1 → S3 루프)

```
앱 실행 → 산책 시작(S1) → [매칭 있음/없음] → 산책 완료 → 기록 남기기(S3-F-10) → 저장
         ↑______________________________________________________________|
```

매칭 성사 여부와 무관하게 기록이 남는다는 점이 핵심 —
앱 사용이 매칭 결과에 종속되지 않고, 기록 루틴으로 독립된다.

| 항목 | 필수 여부 |
|---|---|
| 이름 | 필수 |
| 사진 | 필수 |

| 나이 | 필수 |
| 성별 | 필수 |
| 크기 | 필수 |
| 중성화 여부 | 선택 |
| 성격 태그 | 필수 (최소 1개) |
| 사회성 | 선택 |
| 활동량 | 선택 |
| 산책 스타일 | 선택 |
| 선호 친구 유형 | 선택 |
| 주의사항 | 선택 |

#### 성격 태그 목록 (예시)

`사람 좋아함` `강아지 좋아함` `소심함` `활발함` `천천히 걷기 선호` `뛰는 산책 선호` `대형견 무서워함` `짖음 있음`

## 5. S1. 산책 매칭 시스템

> 강아지 전용. 이 서비스의 핵심 진입점이자 S2(관계), S3(기록)의 출발점.

### F-01. 실시간 산책 지도

**목적:** 지금 근처에 함께 산책 가능한 반려견이 있는지 확인한다.

#### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 산책 시작 버튼 | 위치 공유 시작, 지도 활성화 |
| 현재 위치 표시 | 내 위치 마커 |
| 근처 강아지 아이콘 | 산책 중인 다른 사용자 표시 |
| 거리 표시 | 아이콘 옆 대략적 거리 (예: 200m) |
| 기본 프로필 미리보기 | 아이콘 탭 시 이름·견종·성격 태그 요약 |
| 필터 | 크기·성격 등으로 필터링 |

#### 기획 원칙

- 정확한 좌표를 그대로 노출하지 않고, 대략적 위치 또는 일정 반경 내 아이콘으로 표시한다
- 산책 종료 시 즉시 지도에서 사라진다

---

### F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 반려견과 즉시 산책 연결을 시도한다.

#### 사용자 플로우

```
프로필 확인
  → [같이 산책하기] 버튼 클릭
  → 상대방에게 요청 알림 발송
  → 상대방 수락 또는 거절
  → 수락 시 매칭 성사 (F-04 시작)
```

#### 요청 상태

| 상태 | 설명 |
|---|---|
| 요청 대기 | 상대방 응답 전 |
| 수락 | 매칭 세션 생성 |
| 거절 | 요청 종료, 상대방에게 노출 최소화 |
| 시간 만료 | N분 내 응답 없으면 자동 만료 (시간 TBD) |
| 취소 | 요청자가 직접 취소 |

#### 기획 원칙

- 요청은 가볍고 부담이 적어야 한다
- 거절은 상대방에게 부정적으로 노출되지 않도록 설계한다

---

### F-04. 매칭 세션

**목적:** 함께 산책하는 동안의 상태를 관리하고, 종료 후 로그를 생성한다.

#### 세션 정보

| 항목 | 설명 |
|---|---|
| 참여 사용자 | 요청자 + 수락자 |
| 참여 반려견 | 양측 반려견 정보 |
| 시작 시간 | 수락 시점 |
| 종료 시간 | 산책 종료 버튼 또는 자동 종료 |
| 산책 위치 범위 | 대략적 산책 구역 |
| 동행 시간 | 세션 지속 시간 |
| 동행 거리 | 이동 거리 (선택) |

#### 세션 중 제공 기능

| 기능 | 설명 |
|---|---|
| 합류 안내 | 만남 장소 안내 |
| 간단 메시지 | 짧은 텍스트 또는 이모지 |
| 산책 종료 | 세션 종료 → F-05 로그 생성 → F-10 기록 화면 자동 진입 |
| 신고 / 차단 | 부적절한 사용자 신고 |
| 매칭 취소 | 세션 중 이탈 |

---

### F-05. 매칭 로그

**목적:** 일회성 만남을 데이터로 축적하고, S2(관계)와 S3(기록)의 기반 데이터를 만든다.

#### 로그 항목

| 항목 | 저장 방식 |
|---|---|
| 함께 산책한 강아지 | 자동 |
| 산책 날짜 / 시간 / 거리 | 자동 |
| 만남 횟수 | 누적 카운트 |
| 사진 | 사용자 업로드 (선택) |
| 산책 후 피드백 | 사용자 입력 (선택) |

#### 로그 활용처

| 활용처 | 시스템 |
|---|---|
| 친구 요청 활성화 조건 | S2 |
| 친밀도 점수 입력값 | S2 |
| 다이어리 기본 데이터 | S3 |
| Pet Space 피드 데이터 | S4 |

## 6. S2. 소셜/관계 시스템

> S1 매칭 이력이 없으면 진입 불가. 우연한 만남을 지속적 관계로 전환하는 시스템.

### F-06. 친구 시스템

**목적:** 우연한 산책 만남을 지속적인 관계로 전환한다.

**활성화 조건:** 1회 이상 매칭 성공한 상대에게만 친구 요청 가능

#### 주요 기능

| 기능 | 설명 |
|---|---|
| 친구 요청 / 수락 / 거절 | 기본 친구 관계 관리 |
| 친구 목록 | 친구 목록 + 마지막 산책 정보 |
| 친구별 산책 기록 | 함께한 산책 히스토리 |
| 친구 알림 설정 | 알림 ON/OFF 개별 설정 |
| 친구 차단 / 삭제 | 관계 종료 |

---

### F-07. 친밀도 시스템

**목적:** 반복 만남을 유도하고, 강아지 간 관계 형성을 서비스의 핵심 재미 요소로 만든다.

> 명칭 후보: 친밀도 / 산책 케미 / 친구 레벨 / 펫 케미 지수

#### 점수 변동 기준

| 상승 요인 | 하락/정체 요인 |
|---|---|
| 같은 강아지와 반복 산책 | 장기간 미만남 |
| 친구 요청 수락 | 반복적인 요청 거절 |
| 산책 후 긍정 피드백 | 산책 후 부정 피드백 |
| 일정 시간 이상 함께 산책 | 친구 삭제 / 차단 |
| 호출 후 실제 산책 성사 | — |
| 방(Room)에 공유 기록 추가 | — |

#### 기획 원칙

- 하락은 사용자에게 직접 강조하지 않는다
- 부정 피드백은 상대방에게 공개하지 않는다

---

### F-08. 산책 호출

**목적:** 한 번 만난 상대와 다시 만날 수 있는 구조를 만든다.

#### 단계별 기능

| 단계 | 조건 | 가능한 기능 |
|---|---|---|
| **1단계: 산책 친구** | 1회 매칭 또는 친구 수락 | "같이 산책할래요?" 알림 / 과거 산책 로그 확인 |
| **2단계: 자주 만나는 친구** | 반복 산책 + 긍정 피드백 + 상호 친구 | 상대 산책 시작 시 알림 (양측 동의) / 대략적 산책 구역 확인 |
| **3단계: 베스트 산책 메이트** | 높은 친밀도 + 반복 산책 + 상호 동의 | 정기 산책 제안 / 빠른 호출 / 그룹 산책 초대 / AI 타이밍 추천 |

## 7. S3. 기록 시스템

> S1 산책 완료가 진입점이지만, 매칭 없이도 사용 가능. F-10이 콘텐츠 생산 레이어이고 F-11은 닫힌 공유 레이어다.

### F-10. 산책 기록 다이어리 (콘텐츠 생산 레이어)

**목적:** 산책 완료 후 기록을 남겨 루틴 형성과 앱 재방문을 유도한다. F-11(방 공유)·F-13(동네 커뮤니티)의 콘텐츠 공급원 역할을 한다.

> 매칭 유무와 무관하게 산책 완료 기록은 항상 남길 수 있다.

#### 기록 기본 포맷

기본 포맷은 **영상** (산책 중 촬영하거나, 사진들로 자동 구성).
영상/사진 위에 텍스트, 스티커, 그림을 얹어 꾸밀 수 있다 (다꾸 감성).
미디어 없이 텍스트만으로도 기록 가능 (진입 장벽 최소화).

#### 기록 진입 플로우

```
산책 종료 버튼 클릭
  → 기록 남기기 화면 자동 진입
  → 영상/사진 업로드 또는 촬영 (선택)
  → 텍스트·스티커·그림 등으로 꾸미기 (선택)
  → 공개 범위 설정
       ├─ 일기 (나만 보기)
       ├─ F-11 방(Room) 공유
       └─ F-13 동네 공유
  → 저장
```

#### 기록 포함 정보

| 항목 | 저장 방식 |
|---|---|
| 산책 날짜 / 시간 / 거리 | 자동 |
| 함께한 반려견 (매칭이 있었을 경우) | 자동 |
| 영상 / 사진 | 사용자 업로드 또는 촬영 |
| 텍스트, 스티커, 그림 | 사용자 입력 |
| 공개 범위 | 사용자 선택 |

#### 주요 UI

| 요소 | 설명 |
|---|---|
| 기록 꾸미기 에디터 | 영상/사진 위에 텍스트·스티커 추가 |
| 캘린더 뷰 | 날짜별 산책 기록 확인 |
| 산책 스탯 | 누적 거리, 산책 횟수, 연속 일수 |

#### 기획 원칙

- 기록을 남기는 행위가 산책 루틴의 마무리가 되도록 UX 설계 (열린 루프 닫기)
- 다꾸 감성 + 운동 기록 앱 스탯 구조

### F-11. 방(Room) 기반 공유 기록 (닫힌 공유 레이어)

**목적:** F-10에서 생산된 기록을 특정 친구 그룹과 공유하는 닫힌 공간을 만든다.

> F-10 없이는 방에 올릴 콘텐츠가 없다.

#### 방 개요

- 방은 특정 반려견 조합 또는 친구 그룹 단위로 생성
- F-10에서 꾸며둔 개인 기록을 방에 업로드하는 방식으로 연결
- 방 개설 조건: 1회 이상 매칭 성공 + 친구 관계인 상대와만 생성 가능

#### 주요 기능

| 기능 | 설명 |
|---|---|
| 방 생성 | 친구 관계 기반, 반려견 조합 지정 |
| 기록 업로드 | F-10 기록을 방에 공유 |
| 방 히스토리 | 함께한 산책 기록 타임라인 |
| 방 참여 관리 | 초대 / 나가기 |

## 8. S4. Pet Space 시스템

> F-02 프로필 + S1 매칭 이력 + S3 기록이 반려동물의 공간을 채워준다.

### F-12. Pet Space

**목적:** 반려동물이 자신의 디지털 공간을 갖고, 기록과 관계가 반려동물 중심으로 쌓이도록 한다.

#### Pet Space 구성

| 구성 요소 | 채워지는 데이터 |
|---|---|
| 피드 | S3 산책 기록, S1 매칭 기록 |
| 반려동물 자기소개 | F-02 프로필 (의인화 톤) |
| 공간 꾸미기 | 주인이 직접 꾸밈 (배경, 소품, 스티커) |
| 방문 | S2 친구 관계의 다른 반려동물 Pet Space |

#### 의인화 UX

```
일반적인 반려동물 앱: "보미, 말티즈, 3살"
Pet Space 의인화 UX: "안녕, 나는 보미야. 느린 산책이 제일 좋아."
```

의인화는 전체 UX의 톤앤매너로 자연스럽게 스며들어야 한다.
억지스러운 의인화는 오히려 어색하므로 강도 조절이 중요하다.

## 9. S5. 커뮤니티 시스템

> S3 기록(F-10)이 콘텐츠 공급원. S1 매칭 이력이 있는 사용자 기반으로 신뢰 커뮤니티 형성.

### F-13. 동네 커뮤니티 (열린 공유 레이어)

**목적:** F-10 기록을 동네 반려인과 공유하고, 지역 반려인 네트워크를 형성한다.

> F-10 없이는 피드를 채울 콘텐츠가 없다.

#### 주요 기능

| 기능 | 설명 |
|---|---|
| 동네 피드 | 같은 동네 반려동물 기록을 피드 형태로 확인 |
| 동네 반려인 발견 | 자주 산책하는 근처 반려인 탐색 |
| 게시글 작성 | 동네 공개 기록 또는 자유 게시글 |
| 댓글 / 반응 | 커뮤니티 상호작용 |
| 모임 제안 | 동네 그룹 산책 또는 모임 공지 |

#### 기획 원칙

- 동네 기반 게시글은 정확한 위치를 노출하지 않는다
- 초반 피드 공백 해소: 산책 기록 자동 공유 옵션 (사용자 선택)

---

### F-14. 니치 커뮤니티 (종별 커뮤니티)

**목적:** 반려동물 종별 관심사 기반 커뮤니티를 제공한다.

#### 종별 커뮤니티

| 반려동물 | 커뮤니티 주요 콘텐츠 | 산책 매칭 확장 가능성 |
|---|---|---|
| 고양이 | 품종별 건강 정보, 실내 환경 구성 | 하네스 산책 유저 한정 |
| 토끼 | 식이/질병 정보, 운동공간 공유 | 야외 놀이 모임 (매우 니치) |
| 앵무새/조류 | 훈련·언어 발달, 품종별 특성 | 공원 외출 모임 가능성 |
| 파충류 | 사육 환경, 먹이 정보 | 기록 레이어만 유효 |
| 소동물 | 수명·질병, 케어 정보 | 기록 레이어만 유효 |

#### 확장 원칙

- F-13 동네 커뮤니티 활성화 이후 단계적으로 확장
- F-02 프로필에 등록된 반려동물 종 기반으로 자동 추천
