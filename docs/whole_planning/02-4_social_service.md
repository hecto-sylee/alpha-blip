# 02-4. 소셜/관계 서비스

> 관련 기능: F-06 (친구 시스템), F-07 (친밀도 시스템), F-08 (산책 호출), F-11 (방 기반 공유 기록)
> MVP 포함: F-06 ✅ Phase 2 / F-07, F-08 Phase 2~3 / F-11 Phase 3

---

## 서비스 개요

우연한 산책 만남을 지속적인 관계로 전환하는 서비스.
매칭 이력을 기반으로 친구를 맺고, 반복 만남을 통해 친밀도를 쌓으며, 다시 만날 수 있는 호출 구조를 제공한다.
Phase 3에서는 방(Room) 기반 공유 기록을 통해 친구 관계가 공동의 기록 공간으로 확장된다.

---

## F-06. 친구 시스템

**목적:** 우연한 산책 만남을 지속적인 관계로 전환한다.

**활성화 조건:** 한 번 이상 매칭이 성공한 사용자에게만 친구 요청 가능

### 주요 기능

| 기능 | 설명 |
|---|---|
| 친구 요청 / 수락 / 거절 | 기본 친구 관계 관리 |
| 친구 목록 | 친구 목록 + 마지막 산책 정보 |
| 친구별 산책 기록 | 함께한 산책 히스토리 |
| 친구 알림 설정 | 알림 ON/OFF 개별 설정 |
| 친구 차단 / 삭제 | 관계 종료 |

### 기획 원칙

아무에게나 친구 요청을 보내는 구조가 아니라, 실제로 한 번 이상 산책한 상대를 중심으로 관계를 형성한다.

---

## F-07. 친밀도 시스템

**목적:** 반복 만남을 유도하고, 강아지 간 관계 형성을 서비스의 핵심 재미 요소로 만든다.

> 명칭 후보: 친밀도 / 산책 케미 / 친구 레벨 / 펫 케미 지수

### 점수 변동 기준

| 상승 요인 | 하락/정체 요인 |
|---|---|
| 같은 강아지와 반복 산책 | 장기간 미만남 |
| 친구 요청 수락 | 반복적인 요청 거절 |
| 산책 후 긍정 피드백 | 산책 후 부정 피드백 |
| 일정 시간 이상 함께 산책 | 친구 삭제 |
| 호출 후 실제 산책 성사 | 차단 |
| 방(Room)에 공유 기록 추가 | — |

### 기획 원칙

- 하락은 사용자에게 직접적으로 강조하지 않는다
- 부정 피드백은 상대방에게 공개하지 않는다

---

## F-08. 산책 호출 기능

**목적:** 처음 만난 상대와 다시 만날 수 있는 구조를 만든다.

### 단계별 기능

| 단계 | 조건 | 가능한 기능 |
|---|---|---|
| **1단계: 산책 친구** | 1회 매칭 또는 친구 수락 | "같이 산책할래요?" 알림 전송 / 과거 산책 로그 확인 |
| **2단계: 자주 만나는 친구** | 반복 산책 + 긍정 피드백 + 상호 친구 관계 | 상대 산책 시작 시 알림 수신 (양측 동의 필요) / 대략적 산책 구역 확인 |
| **3단계: 베스트 산책 메이트** | 높은 친밀도 + 반복 산책 + 상호 동의 | 정기 산책 제안 / 빠른 호출 / 그룹 산책 초대 / AI 기반 산책 타이밍 추천 |

## F-11. 방(Room) 기반 공유 기록 (Phase 3)

**목적:** 특정 반려견 조합 또는 친구 그룹 단위로 기록을 모아 공유하는 공동 공간을 만든다.

### 방 개요

- 방은 특정 반려견 조합 또는 친구 그룹 단위로 생성
- 방 안에서 매칭 기록, 산책 완료 기록 등을 함께 쌓음
- 개인 다이어리(F-10)에서 꾸민 기록을 방에 업로드하는 방식으로 연결

### 주요 기능

| 기능 | 설명 |
|---|---|
| 방 생성 | 친구 관계 기반, 반려견 조합 지정 |
| 기록 업로드 | 개인 다이어리 기록을 방에 공유 |
| 방 히스토리 | 함께한 산책 기록 아카이브 |
| 방 참여 관리 | 초대 / 나가기 |

### 방 개설 조건

한 번 이상 매칭 성공 + 친구 관계인 상대와만 방 생성 가능

### 활용 시나리오

- 자주 같이 산책하는 친구 반려견과의 기록 모아보기
- 그룹 산책 히스토리 아카이브
