# 02-3. 매칭 서비스

> 소속 시스템: **S1. 산책 매칭**
> 관련 기능: F-03 (같이 산책하기 요청), F-04 (매칭 세션), F-05 (매칭 로그)
> Phase: ✅ Phase 1

---

## 시스템 개요

S1 산책 매칭 시스템의 핵심 흐름.
지도에서 발견한 상대에게 요청을 보내고, 수락 시 세션을 생성하며, 종료 후 로그를 남긴다.
매칭 로그(F-05)는 S2(소셜/관계)의 친구 요청 조건 및 친밀도 입력값이 되고,
산책 종료 시 S3(기록)의 F-10 다이어리로 자동 연결된다.

---

## F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 반려견과 즉시 산책 연결을 시도한다.

### 사용자 플로우

```
프로필 확인 (F-02)
  → [같이 산책하기] 버튼 클릭
  → 상대방에게 요청 알림 발송
  → 상대방 수락 또는 거절
  → 수락 시 매칭 성사 (F-04 시작)
```

### 요청 상태

| 상태 | 설명 |
|---|---|
| 요청 대기 | 상대방 응답 전 |
| 수락 | 매칭 세션 생성 |
| 거절 | 요청 종료, 상대방에게 노출 최소화 |
| 시간 만료 | N분 내 응답 없으면 자동 만료 (시간 TBD) |
| 취소 | 요청자가 직접 취소 |

### 기획 원칙

- 요청은 가볍고 부담이 적어야 한다
- 거절은 상대방에게 부정적으로 노출되지 않도록 설계한다

---

## F-04. 매칭 세션

**목적:** 함께 산책하는 동안의 상태를 관리하고, 종료 후 로그를 생성한다.

### 세션 정보

| 항목 | 설명 |
|---|---|
| 참여 사용자 | 요청자 + 수락자 |
| 참여 반려견 | 양측 F-02 반려동물 정보 |
| 시작 시간 | 수락 시점 |
| 종료 시간 | 산책 종료 버튼 또는 자동 종료 |
| 산책 위치 범위 | 대략적 산책 구역 |
| 동행 시간 | 세션 지속 시간 |
| 동행 거리 | 이동 거리 (선택) |

### 세션 중 제공 기능

| 기능 | 설명 |
|---|---|
| 합류 안내 | 만남 장소 안내 |
| 간단 메시지 | 짧은 텍스트 또는 이모지 |
| 산책 종료 | 세션 종료 → F-05 로그 생성 → F-10 기록 화면 자동 진입 |
| 신고 / 차단 | 부적절한 사용자 신고 |
| 매칭 취소 | 세션 중 이탈 |

### 산책 종료 후 플로우

```
산책 종료 버튼 클릭
  → F-05 매칭 로그 자동 저장
  → F-10 기록 남기기 화면 진입 (S3)
  → 저장 완료
  → 친구 요청 제안 (F-06, S2)
```

---

## F-05. 매칭 로그

**목적:** 일회성 만남을 데이터로 축적하고, S2·S3·S4의 기반 데이터를 만든다.

### 로그 항목

| 항목 | 저장 방식 |
|---|---|
| 함께 산책한 반려동물 | 자동 |
| 산책 날짜 / 시간 / 거리 | 자동 |
| 만남 횟수 (상대방별 누적) | 자동 |
| 사진 | 사용자 업로드 (선택) |
| 산책 후 피드백 | 사용자 입력 (선택) |

### 로그 활용처

| 활용처 | 시스템 |
|---|---|
| 친구 요청 활성화 조건 | S2 F-06 |
| 친밀도 점수 입력값 | S2 F-07 |
| 방(Room) 멤버 초대 조건 | S3 F-11 |
| 다이어리 기본 데이터 | S3 F-10 |
| Pet Space 피드 데이터 | S4 F-12 |
