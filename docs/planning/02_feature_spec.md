## F-01. 실시간 산책 지도

**목적:** 지금 근처에 함께 산책 가능한 반려견이 있는지 확인한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 산책 시작 버튼 | 위치 공유 시작, 지도 활성화 |
| 현재 위치 표시 | 내 위치 마커 |
| 근처 강아지 아이콘 | 산책 중인 다른 사용자 표시 |
| 거리 표시 | 아이콘 옆 대략적 거리 (예: 200m) |
| 기본 프로필 미리보기 | 아이콘 탭 시 이름·견종·성격 태그 요약 |
| 필터 | 크기·성격 등으로 필터링 |

### 지도 표시 정보

강아지 이름 / 견종 / 크기 / 나이 / 성격 태그 / 현재 거리 / 산책 상태

### 기획 원칙

- 정확한 좌표를 그대로 노출하지 않고, 대략적 위치 또는 일정 반경 내 아이콘으로 표시한다
- 산책 종료 시 즉시 지도에서 사라진다

## F-02. 반려견 프로필

**목적:** 같이 산책하기 전 반려견의 성향과 기본 정보를 확인한다.

### 프로필 항목

| 항목 | 필수 여부 |
|---|---|
| 이름 | 필수 |
| 사진 | 필수 |
| 견종 | 필수 |
| 나이 | 필수 |
| 성별 | 필수 |
| 크기 | 필수 |
| 중성화 여부 | 선택 |
| 성격 태그 | 필수 (최소 1개) |
| 사회성 | 선택 |
| 활동량 | 선택 |
| 산책 스타일 | 선택 |
| 선호 친구 유형 | 선택 |
| 주의사항 | 선택 |

### 성격 태그 목록 (예시)

`사람 좋아함` `강아지 좋아함` `소심함` `활발함` `천천히 걷기 선호` `뛰는 산책 선호` `대형견 무서워함` `짖음 있음`

## F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 반려견과 즉시 산책 연결을 시도한다.

### 사용자 플로우

```
프로필 확인
  → [같이 산책하기] 버튼 클릭
  → 상대방에게 요청 알림 발송
  → 상대방 수락 또는 거절
  → 수락 시 매칭 성사 (F-04 시작)
```

### 요청 상태

| 상태 | 설명 |
|---|---|
| 요청 대기 | 상대방 응답 전 |
| 수락 | 매칭 세션 생성 |
| 거절 | 요청 종료, 상대방에게 노출 최소화 |
| 시간 만료 | N분 내 응답 없으면 자동 만료 (시간 TBD) |
| 취소 | 요청자가 직접 취소 |

### 기획 원칙

- 요청은 가볍고 부담이 적어야 한다
- 거절은 상대방에게 부정적으로 노출되지 않도록 설계한다

## F-04. 매칭 세션

**목적:** 함께 산책하는 동안의 상태를 관리하고, 종료 후 로그를 생성한다.

### 세션 정보

| 항목 | 설명 |
|---|---|
| 참여 사용자 | 요청자 + 수락자 |
| 참여 반려견 | 양측 반려견 정보 |
| 시작 시간 | 수락 시점 |
| 종료 시간 | 산책 종료 버튼 또는 자동 종료 |
| 산책 위치 범위 | 대략적 산책 구역 |
| 동행 시간 | 세션 지속 시간 |
| 동행 거리 | 이동 거리 (선택) |
| 산책 상태 | 진행 중 / 종료 |

### 세션 중 제공 기능

| 기능 | 설명 |
|---|---|
| 합류 안내 | 만남 장소 안내 |
| 간단 메시지 | 짧은 텍스트 또는 이모지 |
| 산책 종료 | 세션 종료 → 로그 생성 |
| 신고 / 차단 | 부적절한 사용자 신고 |
| 매칭 취소 | 세션 중 이탈 |

---

## F-05. 매칭 로그

**목적:** 일회성 만남을 데이터로 축적하고, 반복 만남과 관계 형성의 기반을 만든다.

### 로그 항목

| 항목 | 저장 방식 |
|---|---|
| 함께 산책한 강아지 | 자동 |
| 산책 날짜 | 자동 |
| 산책 시간 | 자동 |
| 산책 거리 | 자동 (선택) |
| 만남 횟수 | 누적 카운트 |
| 사진 | 사용자 업로드 (선택) |
| 산책 후 피드백 | 사용자 입력 (선택) |

### 로그 활용처

친구 추천 / 친밀도 계산 / 산책 히스토리 / 반복 매칭 유도

---

## F-06. 친구 시스템

**목적:** 우연한 산책 만남을 지속적인 관계로 전환한다.

**활성화 조건:** 한 번 이상 매칭이 성공한 사용자에게만 친구 요청 가능

### 주요 기능

| 기능 | 설명 |
|---|---|
| 친구 요청 / 수락 / 거절 | 기본 친구 관계 관리 |
| 친구 목록 | 친구 목록 + 마지막 산책 정보 |
| 친구별 산책 기록 | 함께한 산책 히스토리 |
| 친구 알림 설정 | 알림 ON/OFF 개별 설정 |
| 친구 차단 / 삭제 | 관계 종료 |

### 기획 원칙

아무에게나 친구 요청을 보내는 구조가 아니라, 실제로 한 번 이상 산책한 상대를 중심으로 관계를 형성한다.

---

## F-07. 친밀도 시스템

**목적:** 반복 만남을 유도하고, 강아지 간 관계 형성을 서비스의 핵심 재미 요소로 만든다.

> 명칭 후보: 친밀도 / 산책 케미 / 친구 레벨 / 펫 케미 지수

### 점수 변동 기준

| 상승 요인 | 하락/정체 요인 |
|---|---|
| 같은 강아지와 반복 산책 | 장기간 미만남 |
| 친구 요청 수락 | 반복적인 요청 거절 |
| 산책 후 긍정 피드백 | 산책 후 부정 피드백 |
| 일정 시간 이상 함께 산책 | 친구 삭제 |
| 호출 후 실제 산책 성사 | 차단 |

### 기획 원칙

- 하락은 사용자에게 직접적으로 강조하지 않는다
- 부정 피드백은 상대방에게 공개하지 않는다

---

## F-08. 산책 호출 기능

**목적:** 처음 만난 상대와 다시 만날 수 있는 구조를 만든다.

### 단계별 기능

| 단계 | 조건 | 가능한 기능 |
|---|---|---|
| **1단계: 산책 친구** | 1회 매칭 또는 친구 수락 | "같이 산책할래요?" 알림 전송 / 과거 산책 로그 확인 |
| **2단계: 자주 만나는 친구** | 반복 산책 + 긍정 피드백 + 상호 친구 관계 | 상대 산책 시작 시 알림 수신 (양측 동의 필요) / 대략적 산책 구역 확인 |
| **3단계: 베스트 산책 메이트** | 높은 친밀도 + 반복 산책 + 상호 동의 | 정기 산책 제안 / 빠른 호출 / 그룹 산책 초대 / AI 기반 산책 타이밍 추천 |

---

## F-09. 개인정보 보호 설정

**목적:** 사용자가 언제, 누구에게, 어느 수준까지 위치를 공유하는지 직접 통제한다.

### 주요 설정 항목

| 설정 | 설명 |
|---|---|
| 위치 공유 ON/OFF | 산책 중에만 위치 공유 |
| 산책 종료 시 자동 비공개 | 종료 즉시 지도에서 제거 |
| 대략적 위치 표시 | 정확한 좌표 대신 반경 내 표시 |
| 친구별 알림 허용 설정 | 특정 친구 알림만 수신 |
| 특정 시간대 알림 제한 | 야간 등 알림 차단 시간 설정 |
| 차단 / 신고 | 특정 사용자 차단 |
| 마지막 위치 자동 만료 | 일정 시간 후 위치 데이터 삭제 |
| 집 주변 위치 노출 방지 | 홈 반경 내 위치 비공개 처리 |
