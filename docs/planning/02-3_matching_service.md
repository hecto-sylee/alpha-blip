# 02-3. 매칭 서비스

> 관련 기능: F-03 (같이 산책하기 요청), F-04 (매칭 세션), F-05 (매칭 로그)
> MVP 포함: ✅ Phase 1

---

## 서비스 개요

지도에서 발견한 상대에게 산책 요청을 보내고, 수락 시 세션을 생성하며, 종료 후 로그를 남기는 서비스.
일회성 만남을 데이터로 축적해 친구 추천과 친밀도 계산의 기반으로 활용한다.

---

## F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 반려견과 즉시 산책 연결을 시도한다.

### 사용자 플로우

```
프로필 확인
  → [같이 산책하기] 버튼 클릭
  → 상대방에게 요청 알림 발송
  → 상대방 수락 또는 거절
  → 수락 시 매칭 성사 (F-04 시작)
```

### 요청 상태

| 상태 | 설명 |
|---|---|
| 요청 대기 | 상대방 응답 전 |
| 수락 | 매칭 세션 생성 |
| 거절 | 요청 종료, 상대방에게 노출 최소화 |
| 시간 만료 | N분 내 응답 없으면 자동 만료 (시간 TBD) |
| 취소 | 요청자가 직접 취소 |

### 기획 원칙

- 요청은 가볍고 부담이 적어야 한다
- 거절은 상대방에게 부정적으로 노출되지 않도록 설계한다

---

## F-04. 매칭 세션

**목적:** 함께 산책하는 동안의 상태를 관리하고, 종료 후 로그를 생성한다.

### 세션 정보

| 항목 | 설명 |
|---|---|
| 참여 사용자 | 요청자 + 수락자 |
| 참여 반려견 | 양측 반려견 정보 |
| 시작 시간 | 수락 시점 |
| 종료 시간 | 산책 종료 버튼 또는 자동 종료 |
| 산책 위치 범위 | 대략적 산책 구역 |
| 동행 시간 | 세션 지속 시간 |
| 동행 거리 | 이동 거리 (선택) |
| 산책 상태 | 진행 중 / 종료 |

### 세션 중 제공 기능

| 기능 | 설명 |
|---|---|
| 합류 안내 | 만남 장소 안내 |
| 간단 메시지 | 짧은 텍스트 또는 이모지 |
| 산책 종료 | 세션 종료 → 로그 생성 |
| 신고 / 차단 | 부적절한 사용자 신고 |
| 매칭 취소 | 세션 중 이탈 |

---

## F-05. 매칭 로그

**목적:** 일회성 만남을 데이터로 축적하고, 반복 만남과 관계 형성의 기반을 만든다.

### 로그 항목

| 항목 | 저장 방식 |
|---|---|
| 함께 산책한 강아지 | 자동 |
| 산책 날짜 | 자동 |
| 산책 시간 | 자동 |
| 산책 거리 | 자동 (선택) |
| 만남 횟수 | 누적 카운트 |
| 사진 | 사용자 업로드 (선택) |
| 산책 후 피드백 | 사용자 입력 (선택) |

### 로그 활용처

친구 추천 / 친밀도 계산 / 산책 히스토리 / 반복 매칭 유도
