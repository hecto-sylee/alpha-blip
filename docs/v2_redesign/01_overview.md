# 01 · 전체 정리 — v2 재설계 기능 명세

> 손그림 5장(홈 지도 / 카메라 촬영 / 기록 page / 펫일기 표시 / 펫일기 작성)을 구조화한 **기능 단일 명세**.
> 운영 규칙·소유권·머지는 [`00_orchestration.md`](00_orchestration.md). 화면별 구현은 `10~16_W*.md`.

## 1. 큰 그림 — 상태머신

```
        앱 진입(인증·펫 보유)
              │
              ▼
        ┌───────────────┐  본인 강아지 탭 → [산책하기]
        │  #/home (지도) │ ─────────────────────────────┐
        │  idle: 모두 표시│                               │
        │  본인=중앙 빨강 │  타 강아지 탭 → [같이 산책하기] │
        └───────────────┘ ──────────────┐               │
                                         ▼               ▼
                              ┌────────────────┐  ┌──────────────┐
                              │ #/matching/:id │  │   #/walk     │
                              │  산책 매칭중    │  │  산책 중(혼자)│
                              │  본인+상대만    │  └──────┬───────┘
                              │  발자국 트래킹  │         │
                              │  [매칭 성공]────┼────────▶│ (매칭 산책도 동일 산책중 UI)
                              └────────────────┘         │
                                                         │ 상단 퀘스트박스 / 좌하단 촬영 / 우하단 종료
                                          ┌──────────────┼───────────────┐
                                          ▼              ▼               ▼
                                   퀘스트 박스 탭   일반 촬영 버튼     통화종료 버튼
                                          │              │               │
                                          ▼              ▼               ▼
                                  #/camera?mission   #/camera        산책 종료 상태
                                          └──────┬───────┘               │
                                  촬영 후(미종료)  │  클립 누적             ▼
                                  → #/walk 복귀 ◀─┘             기록 생성 → #/diary
```

핵심: **#/home(idle 지도)** 와 **#/walk(산책 중)** 는 다른 상태다.
- idle: 산책 세션 없음, 주변 모두 보임, 내 마커는 GPS 위치(빨강).
- 산책 중: active walk session, 퀘스트/촬영/종료 HUD. 매칭이면 동일 UI + match_session 연결.

## 2. 공통 UI

- **하단 탭 3개**: 기록 / 홈 / 마이 (좌·중·우). 라우트·아이콘은 [`00 §2`](00_orchestration.md).
- **랭킹 시스템 전부 삭제 + 방 기능 전면 삭제(프론트 숨김)**:
  - 탭바에서 `방`·`랭킹` 제거(현재 index.html 5탭 → 3탭).
  - `screens/my.js`의 "업적 / 내 방" 링크 제거, `screens/diary.js`의 "방" 버튼 제거.
  - 기록 생성 시 공개범위 `방 공유` 옵션 제거(항상 `diary`).
  - 백엔드 라우터/모델/데모는 **그대로**(복구 보존). 상세 `10_W0_foundation.md`.

## 3. 화면별 요구 (손그림 → 요구사항)

### 3.1 [home] 지도 메인 — *이미지 1* (담당: W1, 라우트 `#/home`)
- 앱 켜자마자 지도(기존 `/walk`의 지도 레이어)가 보인다. 단 **idle 모드**(산책 세션 미시작).
- **(0)** 지도 위 "오늘의 퀘스트" 배너 **삭제**(현 `walk.js`의 `#quest-banner`). 퀘스트는 산책중으로 이동.
- **(1)** 축척 **확대**(zoom ↑, 예: 15→17 근처), **본인**이 화면 **중앙에 빨간색 마커**로.
- **(2)** 주변 사용자 마커는 **강아지 캐릭터만** 노출(이름/거리 메타 칩 제거 — 깔끔한 강아지 핀).
- **(3)** 마커 탭 시 **가운데 팝업(center modal)** 으로 프로필 + CTA(현 바텀시트 `openPreview` → center modal로 교체).
- **(4)** **타 사용자** 강아지 탭 → 팝업 [같이 산책하기] → `#/matching/:id`.
- **(5)** **본인** 강아지 탭 → 팝업 [산책하기] → 산책 세션 시작 → `#/walk`.

### 3.2 [산책 매칭중] — *신규* (담당: W3, 라우트 `#/matching/:id`)
- 기존 지도 화면 유지. 단 **본인 + 매칭 진행중 상대**만 표시(다른 주변 강아지 숨김).
- **실시간 트래킹**: 양쪽이 가까워지는 과정에서 지나온 자리에 **발자국 아이콘** 흔적을 남김
  (프론트 시뮬레이션: 위치 폴링으로 좌표 받아 발자국 마커 누적).
- 하단에 **[매칭 성공]** 버튼 → 매칭 확정 → `#/walk`(매칭 산책 진입, match_session 연결).

### 3.3 [산책 중] (담당: W2, 라우트 `#/walk`)
- 기존 지도 화면. 매칭/혼자 공통 HUD.
- **(1)** 상단 **투명 레이아웃**으로 **퀘스트 2개**(텍스트 한 줄 박스). **지도를 가리지 않음**.
- **(2)** 퀘스트 텍스트 탭 → `#/camera?mission=<id>` (카메라 상단에 퀘스트 한 줄 표기).
- **(3)** (매칭/혼자 공통) 촬영 후 **퀘스트 완료** 처리, 산책 종료 상태 아니면 `#/walk` 복귀.
- **(4)** **좌하단** 일반 촬영 버튼 → `#/camera`(퀘스트 없이). 촬영 후 미종료면 `#/walk` 복귀.
- **(5)** **우하단** 통화종료(전화 끊기) 버튼 → 산책 종료 상태 → 누적 클립으로 기록 생성 → `#/diary`.

### 3.4 [카메라 촬영] — *이미지 2* (담당: W4, 라우트 `#/camera`)
- **가로(landscape) 전체화면** 촬영.
- 퀘스트 진입(`?mission=`/`?quest=`) 시 상단(이미지의 "얼렁뚱땅 헥토의 하루" 위치)에 **퀘스트 내용 한 줄**.
  일반 진입이면 **미표기**.
- 촬영(클립) → 업로드(`POST /clips/upload`) → `clip_id`를 산책 누적 클립에 추가 → 호출 출처로 복귀
  (미종료 → `#/walk`, 종료상태 → `#/diary`).
- 캡처 메커니즘은 기존 `media.js`(2초 클립) 재사용, 레이아웃만 가로 몰입.

### 3.5 [기록] 탭 — *이미지 3* (담당: W5, 라우트 `#/diary`)
크게 **영상 기록 / 펫일기** 두 영역.
- **(1)** 기존 `/diary` 대수술. **(2)** 방 기능 전면 삭제(공유옵션/방 버튼 제거).
- **(3)** 상단 작은 **둥근 "기록" 칩** 탭 → **캘린더/공유 토글** 표시.
  - 캘린더 선택 → 달력에서 날짜 고르면 해당 날짜 기록으로 이동.
  - **공유는 비활성**(자리만, 토스트 "준비 중" 등).
- **(4)** 바로 하단 **내 기록 영상 / 매칭 상대 기록 영상** 레이아웃.
  - **매칭** 산책: 두 영상(내 것 + 상대) 모두 **동영상 썸네일** 노출(상대는 신규 API로 조회).
  - **혼자** 산책: 내 기록 영상만.
- **(5)** 영상 레이아웃 하단 **펫일기** 섹션. 없으면 "일기가 없어요.", 있으면 *이미지 4* 카드.
  카드 탭 → `#/pet-diary/:id`(상세/편집). 생성 진입 → `#/pet-diary/new`.
- **(6)** **스와이프**로 시간(날짜)별 이동 — 영상 기록과 펫일기가 **동시에** 같은 날짜로 갱신.

### 3.6 [펫일기 작성] — *이미지 5* (담당: W6, 라우트 `#/pet-diary/new`)
- "펫일기 생성" 진입 → **펫 기분(mood)** + **활동 태그**(이미지5: 날씨/사람/식사/이동 카테고리별 아이콘) + **텍스트**.
- 저장 → `POST /pet-diary` → 기록 탭 펫일기 섹션에 *이미지 4* 처럼 표시.

## 4. 신규 백엔드 (확정 결정 반영)

### 4.1 PetDiary (W6)
- 모델 `pet_diaries`: `id, user_id, pet_id, diary_date(Date), mood(str/int), activity_tags(JSON list), text(Text, nullable), created_at`.
- API: `POST /pet-diary` · `GET /pet-diary?from&to`(또는 `?date=`) · `GET /pet-diary/{id}` · `PATCH /pet-diary/{id}` · `DELETE /pet-diary/{id}`.
- `server/models.py`에 모델, `server/schemas.py`에 스키마, `server/api/pet_diary.py` 신설 + `main.py` 라우터 등록.

### 4.2 매칭 상대 기록 조회 (W5)
- `GET /match-sessions/{id}/records` → 참여자 전용. `{ mine:[clip...], partner:[clip...] }`
  (각 record의 클립 메타 + `stream_url`). `matches.py`에 엔드포인트 추가(신규 라우터 X).
- **클립 접근권한**: `clips.py`의 stream 접근이 현재 owner/방멤버만 허용 → **매칭 세션 동행자**도
  상대의 match-linked 클립을 볼 수 있도록 권한 확장 필요(스펙 `15_W5`).

## 6. 결정 · 가정 · 오픈 이슈

**확정(2026-06-25 사용자 답변):**
- D1. 랭킹/방/업적 = **프론트만 숨김**(백엔드 무변경).
- D2. 펫일기 = **신규 PetDiary 모델+API**.
- D3. 매칭 상대 기록 = **신규 API 실제 연동**.
- D4. 발자국 트래킹 = **프론트 시뮬레이션**(백엔드 무변경).

**구현 전제(스펙 작성자가 채택, 리뷰 시 정정 가능):**
- A1. 클립→기록은 **산책 종료 시 1개 Record로 번들**(구 `#/record` 에디터 폐기). [`00 §3.1`]
- A2. 사람 텍스트/기분은 **펫일기로 분리**, Record엔 메모 없음.
- A3. 퀘스트 picker 화면 폐기, **산책 시작 시 자동 확보** + 미션 2개 노출.
- A4. 기록 탭은 **날짜 스코프**(달력/스와이프로 날짜 이동, 영상+펫일기 동시 갱신).
- A5. 카메라는 기존 `media.js` 2초 클립 메커니즘 유지(레이아웃만 가로).

**오픈 이슈(구현 중 확인):**
- O1. `POST /match-requests` requester walk session 도출 방식(idle에서 요청 가능 여부) → W1/W3에서 확인.
- O2. 상대 클립 stream 권한 확장 범위(매칭 동행자 한정) → W5에서 확정.
- O3. 펫일기 ↔ 산책 record 연결 여부(현재 standalone per date+pet). 필요 시 `record_id` FK 추가 검토.
- O4. 발자국 트래킹의 자연스러움(헤드리스엔 GPS 없음) → 데모/시뮬 좌표 보간으로 대체.
