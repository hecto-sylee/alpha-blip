# MVP 기획 개요

> 이 문서 세트는 [whole_planning/04_mvp_scope.md](../whole_planning/04_mvp_scope.md)에서 정의한
> MVP 구현 범위(**S1 산책 매칭 + S3 기록 + 공통**)에 대한 상세 기획서다.
> 전체 서비스 비전은 [whole_planning](../whole_planning/) 참고.

---

## MVP 한 줄 정의

> 산책 중인 강아지를 실시간으로 연결하고, 산책이 끝나면 기록을 남기게 하는 모바일 웹 서비스.

---

## MVP 목표

MVP는 두 가지 핵심 가설을 검증한다.

| # | 가설 | 검증 질문 |
|---|---|---|
| 가설 1 | 사람들은 근처 산책 중인 강아지와 즉흥적으로 매칭해 함께 산책한다 | 실시간 매칭이 실제로 성사되는가? |
| 가설 2 | 산책 후 기록을 남기는 경험이 재방문·루틴 형성으로 이어진다 | 매칭이 없어도 기록을 위해 앱을 켜는가? |

**S1(매칭)** 은 앱으로 사람을 끌어오는 훅, **S3(기록)** 은 사람을 머무르게 하는 루틴.
두 레이어가 함께 작동하는지를 보는 것이 MVP의 핵심이다.

## 구현 범위

| 시스템 | 포함 | 기능 |
|---|---|---|
| 공통 | ✅ | F-02 반려동물 프로필 / F-09 개인정보 보호 설정 (S1·S3 항목만) |
| S1 산책 매칭 | ✅ | F-01 실시간 산책 지도 / F-03 같이 산책하기 요청 / F-04 매칭 세션 / F-05 매칭 로그 |
| S3 기록 | ✅ | F-10 산책 기록 다이어리 / F-11 방(Room) 기반 공유 기록 |
| S2 소셜/관계 | ❌ | MVP 이후 |
| S4 Pet Space | ❌ | MVP 이후 |
| S5 커뮤니티 | ❌ | MVP 이후 |

### MVP에서 비활성화되는 의존 항목

S2·S4·S5를 구현하지 않으므로 일부 옵션은 MVP에서 막는다.

- **F-10 공개 범위**: `일기` / `방 공유`만 (동네 공유 = S5 미구현)
- **F-11 방 초대**: `매칭 이력 초대` + `방 코드 공유`만 (친구 초대 = S2 미구현)
- **F-09 설정**: 위치·기록 공개 범위 항목만 (친구 알림·Pet Space·동네 활동 공개 제외)
- **산책 종료 후 친구 요청 제안**: 미노출 (S2 미구현)

## 타깃 사용자

**1차 타깃:** 도심·주거 밀집 지역에서 강아지를 산책시키는 보호자

- 매일 또는 주기적으로 산책한다
- 산책 중 다른 강아지와 자연스럽게 만나고 싶다
- 산책 기록을 남기고 쌓는 데 흥미가 있다

> MVP는 강아지 보호자에 집중한다. (강아지 외 반려동물 확장은 플랫폼 확장기)

## 제품 형태

| 항목 | MVP |
|---|---|
| 형태 | 모바일 웹 (반응형) |
| 기준 환경 | 모바일 브라우저 (산책 중 사용) |
| 위치 수집 | Browser Geolocation API (`watchPosition`) |
| 기술 스택 개요 | Next.js + 지도 API + Supabase 계열 (상세: [dev/01_tech_stack.md](../dev/01_tech_stack.md)) |

> 웹의 한계: 백그라운드 위치 추적이 약하다. MVP는 **앱 실행(포그라운드) 중 산책**을 전제로 한다.

## 핵심 검증 지표

### S1 산책 매칭

| 지표 | 의미 |
|---|---|
| 산책 시작 클릭률 | 핵심 기능 진입 |
| 위치 공유 허용률 | 사용 가능 사용자 비율 |
| 산책 요청률 / 수락률 | 핵심 행동 전환 + 매칭 품질 |
| 매칭 성공률 | 전체 전환 |

### S3 기록

| 지표 | 의미 |
|---|---|
| 산책 후 기록 작성률 | 루틴 형성 핵심 |
| 혼자 산책 기록 비율 | 매칭 비종속 사용 여부 |
| 방 생성·업로드율 | 공유 레이어 작동 여부 |
| 재방문율 | 지속 사용 여부 |
