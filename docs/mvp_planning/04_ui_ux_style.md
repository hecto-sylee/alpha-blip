# MVP UI/UX 스타일 방향 (모던·모션 중심)

> 방향성 문서. 정확한 컬러·폰트 값은 추후 확정하되, **비주얼·모션 컨셉은 이 문서로 고정**한다.
> 2026 기준 최신 흐름(Material 3 Expressive · Apple Liquid Glass · 목적 있는 모션)을 토대로 한다.
> 확정 필요 항목은 하단 [열린 질문](#열린-질문) 참고.

---

## 1. 컨셉 — 하이브리드

두 무드를 시스템으로 분리하되, 같은 브랜드로 잇는다.

| 레이어 | 무드 | 디자인 베이스 | 키워드 |
|---|---|---|---|
| **지도 / 산책 (S1)** | 실시간·몰입·깊이 | **Liquid Glass 계열** | 반투명 글래스 · 블러 · 굴절 · 깊이감 |
| **기록 / 다이어리 (S3)** | 따뜻·발랄·기록 | **Material 3 Expressive 계열** | 볼드 컬러 · 큰 라운드(squircle) · 가변 타이포 · 탄성 모션 |

> 두 무드는 전략의 두 축(지도=모집 / 기록=발판)과 그대로 맞물린다.
> 지도는 콘텐츠(현실 위치) 위에 떠 있는 글래스 UI, 기록은 콘텐츠(내 산책)를 감싸는 따뜻한 표현형 UI.

### 톤앤매너 키워드

`표현적인(expressive)` `살아있는(motion)` `따뜻한` `깊이감 있는` `부담 없는`

> 토스식 클린 미니멀은 이제 기본기 — 그 위에 **표현력과 모션**을 얹어 최신 감도를 만든다.
> 단, 모션은 장식이 아니라 **상태·구조·시스템 의도를 전달**하는 목적형이어야 한다.

## 2. 비주얼 시스템

### 2-1. 지도 / 산책 — Liquid Glass

- **반투명 글래스 패널** — 지도(현실 콘텐츠) 위 컨트롤·바텀시트는 배경을 비추는 유리 재질
- **블러 + 굴절 + 빛 엣지** — 떠 있는 느낌, 콘텐츠에 집중되도록 동적으로 반응
- 강아지 마커: 글래스 칩 + 사진, 거리 라벨. 탭 시 **글래스 바텀시트**가 부드럽게 떠오름
- 산책 중에도 시인성 보장 — 글래스 위 텍스트는 충분한 대비/그림자

### 2-2. 기록 / 다이어리 — Material 3 Expressive

- **볼드 채도 컬러 + 컬러 테마** — 기록은 따뜻하고 발랄하게
- **큰 라운드(squircle) 셰이프** — 카드·버튼·썸네일이 둥글고 큼직
- **가변 타이포(variable type)** — 큰 제목, 표현적 위계 ("오늘 산책 ●")
- **다꾸 감성** — 스티커·그림·손글씨 포인트 자산 (기록 꾸미기와 직결)
- 캘린더/스탯은 **채워지는 감각**을 컬러·셰이프로 시각화

### 2-3. 공통 토큰 (제안, 값 미확정)

| 토큰 | 방향 | 비고 |
|---|---|---|
| 메인 컬러 | 따뜻한 포인트 1색 (코랄/그린 계열 후보) | 값 미확정 |
| 컬러 시스템 | Expressive 컬러 롤 (primary/secondary/tertiary + container) | M3 기반 |
| 폰트 | 한글 가변 서체 (Pretendard Variable 등 후보) | 값 미확정 |
| 셰이프 | 큰 라운드 (squircle, 16~28px) / 글래스 패널은 연속 곡률 | 영역별 차등 |
| 모드 | 라이트 우선, 다크 모드는 추후 | |

## 3. 모션 원칙 ★

모션이 이 디자인의 핵심 차별점. **스프링(탄성) 기반**으로 살아있게, 단 목적을 가지고.

### 원칙

- **목적형 모션** — 모든 움직임은 상태 변화·관계·시스템 의도를 전달한다 (장식 금지)
- **스프링/탄성** — 선형 이징 대신 스프링 커브로 자연스럽고 쫄깃하게
- **마이크로인터랙션** — 모든 주요 액션에 즉각 피드백 (탭 리플, 성공 바운스, 오류 셰이크)
- **햅틱 동반** — 핵심 순간(요청·매칭 성사·저장)에 햅틱 + 모션 동기화
- **접근성** — `prefers-reduced-motion` 존중, 과한 모션은 자동 축소/대체

### 핵심 모션 (예시)

| 순간 | 모션 | 햅틱 |
|---|---|---|
| 산책 시작 | 버튼 → 지도 전환, 글래스 패널 떠오름 | 가벼운 탭 |
| 마커 탭 | 글래스 바텀시트 스프링 업 | 가벼운 탭 |
| 같이 산책 요청 | 버튼 눌림 + 전송 펄스, 대기 인디케이터 | 미디엄 |
| **매칭 성사** | 풀스크린 축하 모션(컨페티/바운스) | 성공 햅틱 |
| 요청 거절/만료 | 부드러운 페이드 다운 (부정 강조 X) | 없음 |
| 기록 저장 | 카드가 캘린더로 날아가 안착(스프링) | 성공 햅틱 |
| 스탯 증가 | 숫자 카운트업 + 셰이프 채워짐 | 가벼운 탭 |
| 방 새 기록 | 타임라인 상단에 슬라이드 인 | 가벼운 탭 |

## 4. 화면별 적용

### 지도 / 매칭 (SCR-11, 12, 14)

- 풀스크린 지도 + **글래스 컨트롤**(하단 글래스 바)
- 상대 프로필: 글래스 바텀시트로 점진 노출, [같이 산책하기]는 큰 단일 CTA
- 매칭 성사 → **축하 모션**으로 감정 피크 (서비스의 하이라이트 순간)

### 기록 에디터 (SCR-20)

- 산책 종료 직후 **부드럽게 떠오르는 에디터** — "기록 안 함"도 쉽게 빠져나가게
- 미디어 위 스티커·텍스트는 **드래그·탄성**으로 다루는 즐거움
- 빈손 저장 허용 — 자동 정보(거리·시간)만으로도 카드 완성

### 다이어리 / 방 (SCR-21, 25)

- 캘린더가 **채워지는 모션** + 연속 일수·스탯 카운트업 → 쌓이는 감각
- 방 타임라인: 새 기록 슬라이드 인, 누가 올렸는지 명확

## 5. 컴포넌트 패턴

| 컴포넌트 | 패턴 |
|---|---|
| 글래스 바텀시트 | 지도 위 정보 노출 (반투명·블러·스프링 업) |
| Squircle 카드 | 기록·방·프로필 (큰 라운드, 탭 시 바운스) |
| 볼드 타이포 헤더 | 화면 제목은 크게·표현적으로 |
| 단일 CTA | 화면당 1개, 큰 라운드, 눌림 모션 + 햅틱 |
| FAB(기록하기) | 다이어리에서 떠 있는 둥근 액션 |
| 빈 상태 | 항상 다음 행동 유도 + 작은 모션으로 생기 |
| 토스트/피드백 | 스프링 인/아웃, 오류는 셰이크 |

---

## 6. 접근성 / 모바일 우선

- 모바일 웹 우선 (산책 중 한 손 사용) — 주요 액션 하단, 터치 타깃 ≥ 44px
- 글래스 위 텍스트 대비 WCAG AA 보장 (반투명이라도 가독성 우선)
- `prefers-reduced-motion`: 스프링·축하 모션을 페이드/즉시 전환으로 대체
- 야외 밝은 환경 고려한 대비

## 7. 레퍼런스

### 패턴/플로우 갤러리 (실물 앱 참고)

- **Mobbin** — 실물 앱 화면 패턴 라이브러리 (정적 화면 스캔)
- **Appshots** — **모션·전환 중심** 갤러리 (온보딩·권한·매칭 플로우의 타이밍 참고에 최적)
- **Refero** — 최신 제품 UX 패턴
- **Banani** — 무료 앱 UI 레퍼런스

### 플랫폼 디자인 언어

- **Material 3 Expressive** (Android 16) — 기록 레이어 베이스 (볼드 컬러·가변 타이포·탄성 모션)
- **Apple Liquid Glass** (iOS 26) — 지도 레이어 베이스 (반투명·굴절·깊이)

> 국내 레퍼런스(토스/당근/배민 등)는 정보 구조·한글 타이포 기본기로 참고하되,
> 비주얼·모션 감도는 위 최신 플랫폼 언어 기준으로 한 단계 끌어올린다.
