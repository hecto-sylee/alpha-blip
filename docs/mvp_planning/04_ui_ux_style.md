# MVP UI/UX 스타일 방향 (제안)

> ⚠️ 디자인은 아직 확정되지 않았다. 이 문서는 **방향성 제안**이며, 구체 시안·컬러·폰트는 추후 결정한다.
> 결정이 필요한 항목은 문서 하단 [열린 질문](#열린-질문)에 모아둔다.

---

## 1. 디자인 컨셉

MVP는 두 가지 경험이 공존한다.

| 레이어 | 성격 | UX 무드 |
|---|---|---|
| S1 산책 매칭 | 실시간·즉흥·야외 | 빠르고 명확, 한 손 조작, 시인성 |
| S3 기록 | 감성·기록·꾸미기 | 따뜻하고 아기자기, 다꾸 감성 |

> **핵심 긴장:** 산책 중(야외·햇빛·움직임)에는 "기능적 명료함"이, 기록할 때는 "감성적 따뜻함"이 필요하다.
> 두 무드를 같은 브랜드 안에서 자연스럽게 잇는 것이 디자인 과제.

### 톤앤매너 키워드 (제안)

`따뜻한` `아기자기한` `직관적인` `부담 없는` `반려동물 친화적`

### 의인화 톤 (약하게)

전체 서비스 비전에는 반려동물 의인화(Pet Space)가 있으나, **그건 S4(MVP 외)**.
MVP에서는 의인화를 **카피의 톤 수준으로만 약하게** 적용한다. (과하지 않게)

- 예: "오늘 산책 어땠어요?" / "보미의 첫 산책 기록이 쌓였어요"
- 억지 의인화(1인칭 반려동물 말투)는 MVP에서 지양 — S4에서 본격화

## 2. 비주얼 방향 (제안, 미확정)

### 컬러

- 메인 톤: 따뜻한 중성 톤 베이스 + 포인트 1색 (제안: 그린/코랄 계열 — **미확정**)
- 야외 시인성: 지도 화면은 고대비 보장, 포인트 색은 마커·CTA에 집중
- 기록 화면: 부드러운 파스텔/뉴트럴로 콘텐츠(사진·영상)가 주인공이 되게

### 타이포그래피

- 한글 가독성 우선 (Pretendard 등 제안 — 미확정)
- 산책 화면: 큰 글씨·명확한 위계 (걸으면서 봄)
- 기록 화면: 본문 가독성 + 손글씨/라운드 느낌 포인트(다꾸 감성) 여지

### 형태 / 무드

- 라운드 코너, 부드러운 그림자 (아기자기)
- 반려동물 일러스트/스티커 자산 여지 (기록 꾸미기와 연결)

## 3. 화면별 UX 원칙

### 지도 / 산책 (SCR-11, SCR-14)

- **한 손·엄지 조작** 전제 — 주요 액션은 하단에 배치
- [산책 시작]/[산책 종료]/[같이 산책하기]는 크고 명확한 단일 CTA
- 야외 햇빛 고려: 고대비, 충분한 터치 영역(최소 44px)
- 정보 최소화: 산책 중엔 꼭 필요한 것만. 상대 정보는 바텀시트로 점진 노출
- 위치는 항상 "대략적"으로 — 정확 좌표 노출 안 함 (신뢰·안전)

### 요청 / 매칭 (SCR-12, SCR-13)

- 요청은 **원탭, 부담 없이** — 메시지 강제 없음
- 거절·만료는 **부드럽게** — 부정적 표현 지양 ("다음 기회에 만나요" 류)
- 대기 상태는 명확한 피드백(타이머/인디케이터)

### 기록 에디터 (SCR-20)

- 산책 종료 직후 **자연스럽게 진입** — "기록 안 함"도 쉽게 빠져나갈 수 있게
- **빈손 저장 허용** — 텍스트만, 혹은 자동 정보(거리·시간)만으로도 저장 가능
- 꾸미기는 **선택적 즐거움** — 강제하지 않되, 하고 싶으면 풍부하게
- 자동 채워지는 정보(날짜·거리·함께한 강아지)로 작성 부담 ↓

### 다이어리 / 방 (SCR-21, SCR-25)

- 기록이 **쌓이는 감각**을 시각화 (캘린더 채워짐, 연속 일수, 스탯)
- 빈 상태(첫 사용)에 동기 부여 카피 — "첫 산책을 기록해보세요"
- 방은 **함께 쌓는 느낌** — 누가 올렸는지, 새 기록 알림

## 4. 공통 컴포넌트 원칙

| 컴포넌트 | 원칙 |
|---|---|
| 1차 CTA | 화면당 1개, 크고 명확, 하단 고정 |
| 바텀시트 | 점진적 정보 노출 (지도 위 상대 프로필 등) |
| 빈 상태 | 항상 다음 행동을 안내 (빈 화면 방치 금지) |
| 토스트/피드백 | 요청·저장·오류는 즉각 피드백 |
| 권한 요청 | 맥락 안에서 요청 (산책 시작 시 위치 권한) |
| 로딩/실패 | 위치·업로드 실패는 재시도 경로 제공 |

---

## 5. 접근성 / 모바일 우선

- 모바일 웹 우선 설계 (산책 중 한 손 사용)
- 터치 타깃 최소 44×44px
- 텍스트 대비 WCAG AA 지향
- 야외 가독성 (밝은 환경) 고려한 대비
- 다크모드: MVP 필수 아님 (추후)

---

## 6. 카피 톤 가이드

- 친근한 반말~부드러운 존댓말 사이 (브랜드 보이스 확정 필요)
- 반려동물 이름을 호명 ("보미의 산책")해 정서적 연결
- 거절·실패·빈 상태에서 **부정 표현 회피**, 다음 행동 제시
- 안전·위치 관련은 명확하고 신뢰감 있게 (모호하지 않게)
