## 공통

### F-02. 반려동물 프로필

**목적:** 반려동물의 기본 정보·성향을 등록하고, 매칭(S1)과 기록(S3)에서 참조한다.

#### 프로필 항목

| 항목 | 필수 | 입력 형식 | 비고 |
|---|---|---|---|
| 이름 | 필수 | 텍스트 | |
| 사진 | 필수 | 이미지 1장 | 대표 사진 |
| 견종 | 필수 | 선택(검색) | 견종 목록 + "기타" |
| 나이 | 필수 | 숫자(년/월) | |
| 성별 | 필수 | 선택 | 수/암 |
| 크기 | 필수 | 선택 | 소형/중형/대형 |
| 중성화 여부 | 선택 | 토글 | |
| 성격 태그 | 필수(최소 1) | 다중 선택 | 아래 목록 |
| 사회성 | 선택 | 선택 | 높음/보통/낮음 |
| 활동량 | 선택 | 선택 | 높음/보통/낮음 |
| 산책 스타일 | 선택 | 선택 | 천천히/보통/활발 |
| 주의사항 | 선택 | 텍스트 | 자유 입력 |

#### 성격 태그 목록 (예시)

`사람 좋아함` `강아지 좋아함` `소심함` `활발함` `천천히 걷기 선호` `뛰는 산책 선호` `대형견 무서워함` `짖음 있음`

#### 동작 / 규칙

- 1계정 = 1반려동물 (MVP 기준. 다견 가구는 MVP 이후)
- 프로필 미등록 시 산책 시작 불가 → 등록 화면으로 유도
- 프로필은 언제든 수정 가능
- 사진 업로드 실패 시 기본 이미지 사용, 재시도 안내

### F-09. 개인정보 보호 설정

**목적:** 위치·기록 공개 수준을 사용자가 통제한다. (MVP에서는 S1·S3 관련 항목만)

#### MVP 설정 항목

| 설정 | 기본값 | 설명 | 시스템 |
|---|---|---|---|
| 위치 공유 ON/OFF | ON(산책 중) | 산책 중에만 위치 공유 | S1 |
| 산책 종료 시 자동 비공개 | ON | 종료 즉시 지도에서 제거 | S1 |
| 대략적 위치 표시 | ON | 정확한 좌표 대신 반경 내 표시 | S1 |
| 집 주변 위치 노출 방지 | OFF | 홈 반경 지정 시 해당 반경 내 비공개 | S1 |
| 기록 공개 범위 기본값 | 일기 | 새 기록 저장 시 기본 공개 범위 | S3 |
| 차단 / 신고 | — | 특정 사용자 차단·신고 | S1 |

> MVP 제외 항목: 친구별 알림 설정(S2), Pet Space 공개(S4), 동네 활동 공개(S5)

#### 동작 / 규칙

- "대략적 위치 표시"는 MVP에서 **항상 ON 권장** (기본 보호). OFF는 명시적 동의 시에만
- 차단한 사용자는 지도·요청·기록 공유 대상에서 상호 비노출
- 위치 데이터는 산책 종료 후 일정 시간 내 자동 만료

## S1. 산책 매칭

### F-01. 실시간 산책 지도

**목적:** 지금 근처에 함께 산책 가능한 강아지가 있는지 확인한다.

#### 화면 요소

| 요소 | 설명 |
|---|---|
| 산책 시작 버튼 | 위치 공유 시작 + 지도 활성화 (산책 상태 진입) |
| 내 위치 마커 | 현재 위치 (대략) |
| 근처 강아지 아이콘 | 산책 중인 다른 사용자, 대략적 위치 |
| 거리 표시 | 아이콘 옆 대략 거리 (예: 200m) |
| 프로필 미리보기 | 아이콘 탭 → 바텀시트 (이름·견종·크기·성격 태그) |
| 필터 | 크기·성격 태그로 필터 |
| 산책 종료 버튼 | 산책 상태 종료 → 기록 화면 진입 |

#### 동작 / 규칙

- 산책 시작 시 위치 권한 요청. 거부 시 지도 사용 불가 안내
- 위치는 `watchPosition` 기반 주기 업데이트 (포그라운드 한정)
- 다른 사용자 위치는 **대략적 위치(반경 내)** 로만 표시, 정확한 좌표 미노출
- 산책 종료/앱 이탈 시 내 아이콘은 지도에서 즉시 제거
- "산책 중" 상태인 사용자만 지도에 노출

#### 상태

| 상태 | 설명 |
|---|---|
| 산책 전 | 지도 비활성 또는 미리보기, 산책 시작 버튼 노출 |
| 산책 중 | 위치 공유 + 근처 사용자 표시 + 요청 가능 |
| 산책 종료 | 위치 공유 중단, 기록 화면으로 전환 |

#### 예외

- 위치 권한 거부 / 위치 확보 실패 → 안내 + 재시도
- 근처 사용자 0명 → "지금은 근처에 산책 중인 강아지가 없어요" 빈 상태. 혼자 산책 계속 가능

### F-03. 같이 산책하기 요청

**목적:** 지도에서 발견한 상대에게 즉시 산책 연결을 시도한다.

#### 플로우

```
상대 프로필 미리보기
  → [같이 산책하기] 클릭
  → 상대에게 요청 알림 (웹 푸시/인앱)
  → 상대 수락/거절
  → 수락 시 F-04 매칭 세션 시작
```

#### 요청 상태

| 상태 | 설명 | 처리 |
|---|---|---|
| 요청 대기 | 응답 전 | 요청자에게 "대기 중" 표시 |
| 수락 | 매칭 성사 | 양측 F-04 세션 진입 |
| 거절 | 종료 | 요청자에게 부드러운 안내, 상대 노출 최소화 |
| 시간 만료 | N분 무응답 자동 만료 (N=TBD, 기본 2분 제안) | 요청 종료 |
| 취소 | 요청자가 직접 취소 | 요청 종료 |

#### 동작 / 규칙

- 요청은 가볍게 — 메시지 없이 원탭 요청
- 거절은 상대에게 부정적으로 보이지 않게 (사유 비공개, 카운트 비노출)
- 동시 다중 요청 처리: 한 명 수락 시 나머지 자동 만료(제안) — MVP 정책 확정 필요
- 차단 관계는 요청 불가

#### 예외

- 상대가 먼저 산책 종료 → 요청 무효 안내
- 네트워크 지연으로 수락 충돌 → 먼저 처리된 요청 우선, 나머지 만료

### F-04. 매칭 세션

**목적:** 함께 산책하는 동안 상태를 관리하고, 종료 시 로그를 생성한다.

#### 세션 정보

| 항목 | 설명 | 수집 |
|---|---|---|
| 참여 사용자 | 요청자 + 수락자 | 자동 |
| 참여 반려동물 | 양측 F-02 정보 | 자동 |
| 시작 시간 | 수락 시점 | 자동 |
| 종료 시간 | 종료 버튼 또는 자동 종료 | 자동 |
| 동행 시간 | 세션 지속 시간 | 자동 |
| 동행 거리 | 이동 거리 | 자동(선택) |
| 산책 위치 범위 | 대략적 구역 | 자동 |

#### 세션 중 기능

| 기능 | 설명 | MVP |
|---|---|---|
| 합류 안내 | 만남 위치 안내 (대략 위치 + 거리) | ✅ |
| 간단 메시지 | 짧은 텍스트/이모지 | ✅ (경량) |
| 산책 종료 | 세션 종료 → F-05 로그 → F-10 기록 진입 | ✅ |
| 신고 / 차단 | 부적절 사용자 처리 | ✅ |
| 매칭 취소 | 세션 중 이탈 | ✅ |

#### 동작 / 규칙

- 한쪽이 종료하면 세션 종료, 양측 각각 기록 화면으로 진입
- 일정 시간 무활동/위치 끊김 시 자동 종료 (임계값 TBD)
- 세션 종료 시 양측에 동일 매칭 로그 생성

#### 예외

- 한쪽 네트워크 끊김 → 재연결 유예 후 자동 종료
- 합류 실패(만나지 못함) → 종료 시 "산책 안 함" 처리 옵션 (로그 만남 횟수 미집계) — 정책 확정 필요

### F-05. 매칭 로그

**목적:** 매칭 1건을 데이터로 남기고, 기록(S3)의 기본 데이터가 된다.

#### 로그 항목

| 항목 | 저장 |
|---|---|
| 함께 산책한 반려동물 | 자동 |
| 산책 날짜 / 시간 / 거리 | 자동 |
| 만남 횟수 (상대별 누적) | 자동 |
| 사진 | 사용자 업로드(선택) |
| 산책 후 피드백 | 사용자 입력(선택) |

#### MVP 활용처

| 활용처 | 시스템 | MVP |
|---|---|---|
| 다이어리 기본 데이터 | S3 F-10 | ✅ |
| 방(Room) 초대 자격(매칭 이력) | S3 F-11 | ✅ |
| 친구 요청 조건 | S2 | ❌ (미구현) |
| 친밀도 입력값 | S2 | ❌ (미구현) |

#### 동작 / 규칙

- 매칭 세션 종료 시 자동 생성
- 같은 상대와 반복 매칭 시 "만남 횟수" 누적 → F-11 방 초대 자격 판정에 사용

## S3. 기록

### F-10. 산책 기록 다이어리 (콘텐츠 생산 레이어)

**목적:** 산책 완료 후 기록을 남겨 루틴 형성과 재방문을 유도한다. 매칭 유무와 무관하게 항상 기록 가능.

#### 기록 기본 포맷

- 기본 포맷은 **영상** (산책 중 촬영 또는 사진 자동 구성)
- 영상/사진 위에 텍스트·스티커·그림으로 꾸미기 (다꾸 감성)
- **미디어 없이 텍스트만으로도 저장 가능** (진입 장벽 최소화)

#### 진입 플로우

```
산책 종료 (매칭 산책 or 혼자 산책)
  → 기록 남기기 화면 자동 진입
  → 영상/사진 추가 (선택)
  → 꾸미기 (선택)
  → 공개 범위 설정 [ 일기 | 방 공유 ]      ← MVP: 동네 공유 없음
  → 저장
```

#### 기록 항목

| 항목 | 저장 |
|---|---|
| 산책 날짜 / 시간 / 거리 | 자동 (F-05 또는 혼자 산책 세션) |
| 함께한 반려동물 (매칭 시) | 자동 |
| 영상 / 사진 | 업로드/촬영 |
| 텍스트 · 스티커 · 그림 | 입력 |
| 공개 범위 | 선택 (일기 / 방 공유) |

#### 화면 요소

| 요소 | 설명 |
|---|---|
| 기록 에디터 | 미디어 + 텍스트·스티커 편집 |
| 공개 범위 선택기 | 일기 / 방 공유 (MVP) |
| 캘린더 뷰 | 날짜별 기록 확인 |
| 산책 스탯 | 누적 거리·횟수·연속 일수 |

#### MVP 제약

- 공개 범위는 **일기 / 방 공유** 2종 (동네 공유 = S5 미구현)
- "혼자 산책" 진입: 매칭 없이 산책 시작 → 종료 시에도 동일 기록 화면 진입

#### 예외

- 미디어 업로드 실패 → 텍스트만 저장 후 미디어 재첨부 안내
- 저장 중 이탈 → 임시 저장(드래프트) 후 복구 (MVP 권장)

### F-11. 방(Room) 기반 공유 기록 (닫힌 공유 레이어)

**목적:** F-10 기록을 특정 멤버와 공유하는 닫힌 공간. F-10 없이는 콘텐츠가 없다.

#### 방 참여 방법 (MVP)

| 방법 | 설명 | MVP |
|---|---|---|
| 매칭 이력 초대 | 1회 이상 매칭 기록이 있는 상대 초대 | ✅ |
| 방 코드 공유 | 방 코드를 공유해 참여 (앱 외부 공유 가능) | ✅ |
| 친구 초대 | 친구 추가된 상대 초대 | ❌ (S2 미구현) |

#### 주요 기능

| 기능 | 설명 |
|---|---|
| 방 생성 | 누구나 생성, 반려동물 조합/그룹 이름 지정 |
| 멤버 초대 | 매칭 이력 / 방 코드 (MVP) |
| 기록 업로드 | F-10 기록을 방에 공유 |
| 방 히스토리 | 공유 기록 타임라인 |
| 방 참여 관리 | 초대 / 나가기 |
| 방 알림 | 새 기록 업로드 시 참여자 알림 |

#### 동작 / 규칙

- 방 코드: 생성 시 발급, 재발급 가능 (보안상 만료/재발급 정책 TBD)
- 방 멤버는 방 내 공유된 기록만 열람 (개인 일기 기록은 비노출)
- 방장 나가기 시 위임/방 삭제 정책 필요 (MVP 단순화: 방장=생성자 고정, 삭제만 가능)

#### 예외

- 유효하지 않은/만료된 방 코드 → 참여 실패 안내
- 매칭 이력 없는 상대 초대 시도 → 초대 불가 (방 코드 공유 안내)
