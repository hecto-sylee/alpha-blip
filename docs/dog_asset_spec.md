# LetsPaw 강아지 아트 에셋 규격 (핸드오프)

클로드 디자인 앱(이미지 생성)으로 강아지/악세서리 이미지를 뽑아서 넘겨주면,
이 규격대로면 코드에 **수정 없이 그대로** 꽂힌다. 핵심 원칙 2가지:

1. **모든 이미지는 같은 512×512 캔버스 좌표에 정렬** → 악세서리 1세트가 모든 강아지에 얹힘.
2. **투명 배경 PNG (RGBA)** → 배경/그림자/박스 굽지 말 것. 접지그림자는 코드(CSS)에서 넣음.

---

## 0. 컨텍스트 (이게 뭔지)

**결과물 형태 — 낱장 이미지 파일(스프라이트)이다. 프로토타입/슬라이드/웹페이지가 아니다.**
- 디자인 앱은 **각각 독립된 투명 PNG 파일들**을 export 하면 끝. (강아지 1마리 = PNG 1장, 악세 1개 = PNG 1장)
- 화면·페이지·인터랙션을 만들 필요 없음. 이미 돌아가는 웹앱(LetsPaw)에 **에셋으로 꽂힐** 그림 파일만 필요.

**dog asset = 앱/게임용 캐릭터 스프라이트.**
- LetsPaw = 강아지 산책 매칭·기록 모바일형 웹앱. 강아지는 그 앱의 마스코트/아바타 캐릭터.
- 견종별 1마리씩, 포즈 3종(정면/측면/앉기) + 머리/얼굴/몸에 얹는 오버레이 악세서리.
- 한마디로 "귀여운 픽셀 강아지 캐릭터 + 갈아입히는 아이템" 묶음.

**브랜드/디자인 시스템 — 픽셀아트 × 에겐(소프트) 파스텔.**
- UI 폰트: Galmuri(픽셀 폰트) — 앱 UI용. 강아지 이미지엔 글자 없음.
- 테마색: 핑크 `#FF9EBB` / 진핑크 `#FF7FA8` / 크림 `#FFF7EF`·`#FBEEDD` / 테두리 `#E7C9BC`
- 강아지 공통색: 외곽선 코코아 `#3A2E2A`, 볼터치 `#FFAEC9`, 혀 `#FF7FA8`
- 털색 팔레트(COAT): `#FBF6EE #F4E2C6 #E8C39E #E8B96A #F0A24E #E69A4C #C98A52 #9A6A3F #6E4B30 #3B3330 #9FB0BC #6B7884 #F2C9D6 #C9A0E0`
- 무늬/보조색 팔레트: `#FFFFFF #FBF6EE #F6E4BE #2E2622 #9A6A3F #E69A4C #3B3330 #C0CDD6`
- 톤: 채도 낮은 파스텔, 부드럽고 둥글둥글, 무섭지 않게. 전 견종이 "한 작가가 그린 한 세트"처럼 통일.

## 1. 파일 포맷 (받고 싶은 형식)

| 항목 | 값 |
|---|---|
| 확장자 | **PNG (RGBA, 투명)** |
| 캔버스 | **512 × 512 px 정사각**, 배경 완전 투명 |
| 스타일 | **픽셀아트** (도트 그리드 64px → 512px 무보간 확대, 가장자리 선명). 일러스트 카와이도 가능하나 톤 통일 필수 |
| 외곽선 | 진한 코코아브라운 `#3A2E2A` 계열 1톤, 전 견종 동일 두께 |
| 눈 | 큰 둥근 눈 + 흰 하이라이트 점, 볼터치(연핑크) — 전 견종 동일 규칙으로 "한 세트" 느낌 |
| 광원 | 좌상단 고정 (밝은면 위/왼쪽, 그림자 아래/오른쪽) |
| 그림자 | **굽지 말 것** (접지그림자는 코드에서) |
| 여백 | 가장자리 7% 안전영역(약 36px) 비우기 — 잘림 방지 |

### 정렬 기준 (512 캔버스, 정면 포즈 기준 — 악세 정렬의 핵심)
- 좌우 중심: **x = 256** (강아지 정중앙)
- 발 닿는 기준선(접지): **y ≈ 476**
- 전체 키: 약 430px (귀 끝 y≈46 ~ 발 y≈476), 캔버스의 ~84%
- **머리 박스**(모자가 앉는 영역): 중심 (256, 125), 폭 ~250 / 높이 ~230 → 모자 윗변 y≈70
- **눈선**: y ≈ 175
- **목/가슴선**(반다나·넥타이·목도리·망토 시작): y ≈ 315, 폭 ~240
> 측면(side)·앉기(sit) 포즈는 발 기준선 y≈476과 중심 정렬만 맞추면 됨. 악세서리는 **정면(front) 포즈에만** 얹으므로 측면/앉기는 머리 박스를 엄격히 맞출 필요는 없음.

## 3. 우선순위 (이 순서로 주면 단계별로 바로 반영)

- **Tier 1 (최소 — 이것만 와도 앱이 귀여워짐):**
  데모 견종 8 × **front** = 8장
  → `poodle, maltese, shiba, corgi, golden, dachshund, bichon, mix`
  + 악세 9장 = **총 17장**
- **Tier 2:** 위 8견종의 `side` + `sit` (16장) → 포즈토글/달리기 연출
- **Tier 3:** 나머지 8견종 front (8장) → 이후 side/sit
- **(선택) 색 변형:** 같은 견종 다른 털색을 원하면 `dog_{key}_{pose}__{색이름}.png` 로 추가. 안 주면 견종=고정 룩.

## 4. 전달 방법 & 들어갈 위치
- 폴더/zip로 한 번에. 내가 `server/static/img/dogs/`(강아지), `server/static/img/dogs/acc/`(악세)에 배치.
- 파일명만 위 규칙대로면 끝. 코드(렌더러)가 `breed`/`pose`/`equipped`를 파일명으로 매핑.

## 5. 코드 쪽 변경 (참고 — 내가 함)
- `dog/render.js`: 절차생성 → `<img>`/`<svg><image>` 스프라이트 로더로 교체. `dogSVG(appearance,{pose})`가 `dog_{breed}_{pose}.png` + 장착 악세 레이어를 겹쳐 합성.
- 커스터마이저: 색/귀/다리/무늬 미세토글 → **견종 갤러리 + 악세 꾸미기**(+색변형 있으면 색 선택)로 단순화. 포인트→상점→꾸미기 루프는 그대로.
- 공유 PNG·지도 마커·산책/매칭 두 마리: 같은 스프라이트 재사용.
