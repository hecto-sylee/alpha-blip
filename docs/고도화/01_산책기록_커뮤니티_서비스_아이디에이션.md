## 배경 및 목적

기존 서비스의 골자는 **산책 매칭** — 근처의 반려견과 즉흥적으로 같이 산책하는 것.
여기에 더해, 사용자가 **반려동물과 산책을 꾸준히 지속할 수 있도록** 동기를 부여하는 구조가 필요하다.

인간의 운동 기록 앱처럼, 이 앱을 켜고 산책을 시작하는 것 자체가 루틴이 되고,
산책 완료 후 기록이 쌓이는 경험을 통해 **앱 재방문과 산책 습관 형성**을 동시에 유도한다.

---

## 기본 플로우 (신규)

```
앱 실행
  → 산책 시작 (지도 활성화)
  → [매칭 있음] 매칭된 상대와 함께 산책
    or [매칭 없음] 혼자 산책
  → 산책 완료 버튼 클릭
  → 기록 남기기 화면 진입
  → 기록 저장 (일기 / 방 공유 / 동네 공유 선택)
```

> 매칭 여부와 무관하게 **산책 완료 기록은 항상 남길 수 있다.**
> 매칭이 있었다면 상대방 반려견 정보와 함께 기록된다.

---

## 고민 중인 방향 3가지

---

### 방향 1. 혼자만의 기록 (프라이빗 다이어리)

**컨셉:** 산책 기록을 나만 볼 수 있는 일기·다이어리 형태로 저장.

**핵심 UX:**
- 기본 포맷은 **영상** (산책 중 촬영하거나, 사진으로 자동 구성)
- 영상/사진 위에 **텍스트, 스티커, 그림** 등을 얹어 꾸밀 수 있음 (다꾸 감성)
- 완성된 기록은 캘린더 뷰에서 날짜별로 확인 가능

**포함 정보:**
- 산책 날짜 / 시간 / 거리
- 함께한 반려견 (매칭이 있었을 경우)
- 사용자가 직접 남긴 텍스트·미디어

**레퍼런스 감성:** 다꾸(다이어리 꾸미기) + 운동 기록 앱 (산책 스탯)

---

### 방향 2. 방(Room) 기반 공유 기록

**컨셉:** 셋로그처럼 특정 "방"을 중심으로 기록을 모아두고 공유하는 구조.

**핵심 아이디어:**
- 방은 특정 반려견 조합, 또는 친구 그룹 단위로 생성
- 방 안에서 매칭 기록, 산책 완료 기록 등을 함께 쌓음
- **방향 1에서 꾸며둔 개인 기록을 방에 업로드**하는 방식으로 연결 가능
  → 개인 일기(방향 1)가 콘텐츠 생산 도구 역할을 하게 됨

**활용 시나리오:**
- 자주 같이 산책하는 친구 반려견과의 기록 모아보기
- 그룹 산책 히스토리 아카이브

**레퍼런스 감성:** 셋로그 방 구조 + 공유 앨범

**열린 질문:**
- 방 개설 조건은? (친구 관계 필요? 특정 횟수 이상 매칭?)
- 방이 너무 많아지면 관리 복잡도 올라가는 문제

---

### 방향 3. 동네 단위 커뮤니티

**컨셉:** 당근처럼 동네를 기반으로 산책/매칭 기록을 다른 사용자들과 공유.

**핵심 아이디어:**
- 같은 동네 반려견 산책 기록을 피드 형태로 볼 수 있음
- 동네 내 자주 산책하는 반려견·사용자를 발견하는 창구

**불확실한 점:**
- 유저에게 충분한 흥미 포인트가 되는지 불분명
- 위치 기반 커뮤니티 특성상 개인정보 민감도 높음 ([02-5 개인정보/설정 서비스](../planning/02-5_privacy_service.md)와 충돌 가능)
- 콘텐츠 생산량이 초반에 확보되지 않으면 썰렁한 피드 문제

**결론:** 현 시점에서는 보류. 유저 반응 데이터 쌓인 후 재검토.

---

## 방향 간 관계 정리

```
방향 1 (개인 다이어리)
  └─ 콘텐츠 생산 → 방향 2 (방 공유)로 업로드 가능
                  → 방향 3 (동네 커뮤니티)로 공유 가능 (선택)
```

세 방향은 독립적이지 않고, **방향 1이 기반 콘텐츠 생산 레이어**가 되고
방향 2/3는 그 위에 올라오는 공유 레이어로 구성할 수 있다.

## 기존 기능과의 연결점

| 기존 기능 | 연결 방식 |
|---|---|
| F-04 매칭 세션 | 산책 완료 시 기록 화면으로 자동 연결 |
| F-05 매칭 로그 | 기록의 기본 데이터 (날짜/시간/상대방)로 활용 |
| F-06 친구 시스템 | 방(Room) 개설 기준, 방 공유 대상 |
| F-07 친밀도 시스템 | 꾸준한 기록 → 친밀도 상승 요소로 연결 가능 |
