## 서비스 미션

> **"내 반려동물이 더 풍요로운 하루를 보낼 수 있도록"**

반려동물의 행복을 중심에 두는 서비스.
강아지에게는 산책과 사회화가 그 답이고, 고양이에게는 자극과 활동 기록이 그 답이다.
도달하려는 목표는 같지만 수단은 동물마다 다르다.

---

## 왜 강아지부터인가

이 서비스가 강아지로 시작하는 데는 구조적인 이유가 있다.

| 조건 | 강아지 | 타 반려동물 |
|---|---|---|
| 매일 외출 필요 | ✅ | 대부분 ❌ |
| 다른 동물과의 만남을 즐김 | ✅ (품종·성향에 따라) | 드묾 |
| 시간·장소 기반 즉흥 매칭 | ✅ | ❌ |

**→ 산책 매칭은 강아지에 특화된 기능으로 유지한다.**

그러나 "기록을 남기고, 커뮤니티를 통해 연결된다"는 레이어는 모든 반려동물에게 유효하다.

---

## 확장 구조

```
[레이어 1] 산책 매칭          →  강아지 전용 (핵심, 깊게)
[레이어 2] 활동 기록 / 다이어리 →  모든 반려동물 (pet-agnostic)
[레이어 3] 커뮤니티            →  동물 종류별 니치 커뮤니티로 분화
```

레이어 1은 강아지 사용자를 앱으로 끌어오는 핵심 훅.
레이어 2·3은 강아지 외 반려인들도 앱을 사용하게 되는 확장 경로.

---

## 니치 커뮤니티 확장 가능성

반려동물 커뮤니티 특성상, 동물 종별로 관심사와 페인 포인트가 완전히 다르다.
동네 단위 커뮤니티(→ 견주 커뮤니티)에서 출발해,
활성화 이후 종별 니치 커뮤니티로 확장하는 경로를 그릴 수 있다.

| 반려동물 | 커뮤니티 니즈 | 산책 매칭 확장 가능성 |
|---|---|---|
| 고양이 | 품종별 건강 정보, 실내 환경 구성 | 하네스 산책 유저 한정 가능 |
| 토끼 | 식이/질병 정보, 운동공간 공유 | 야외 놀이 모임 (매우 니치) |
| 앵무새/조류 | 훈련·언어 발달, 품종별 특성 | 공원 외출 모임 가능성 있음 |
| 파충류 | 사육 환경, 먹이 정보 | 매칭 없음, 기록 레이어만 유효 |
| 소동물 (햄스터, 기니피그 등) | 수명·질병, 케어 정보 | 매칭 없음, 기록 레이어만 유효 |

## 기존 커뮤니티와의 경쟁 구도

### 주요 기존 서비스

- **네이버 카페** (견종별, 동물 종별 카페): 정보 공유 특화, 이미 대규모 커뮤니티 형성
- **당근마켓 동네생활**: 위치 기반, 가볍고 즉흥적인 공유
- **인스타그램 펫 계정**: 감성 기록, 팔로워 기반 소통
- **동물 전문 앱** (포동, 핏펫 등): 건강 기록, 병원 예약 등 기능 특화

### 기존 커뮤니티(특히 네이버 카페)의 페인 포인트

이 서비스가 공략할 수 있는 실제 불편함들:

| 페인 포인트 | 현상 | 이 서비스가 줄 수 있는 것 |
|---|---|---|
| **내 동물 맥락 없음** | 질문할 때마다 "품종이 뭔가요?", "나이가 어떻게 되나요?" 를 반복해야 함. 커뮤니티는 내 동물을 모른다. | 프로필 기반 → 내 반려동물을 알고 있는 상태에서 연결 |
| **정보 파편화·매몰** | 몇 년 전 게시글이 상단에 뜨거나, 찾고 싶은 정보가 글 더미에 묻혀있음 | 기록과 커뮤니티가 연결된 구조, 피드 흐름 |
| **온라인에서 오프라인으로 연결 안 됨** | 같은 동네, 같은 품종 견주인데 만날 방법이 없음 | 동네 기반 매칭·모임 연결 가능 |
| **텍스트 중심, 낡은 UX** | PC 시대 기반 UI, 모바일 최적화 부족, 사진·영상 공유 불편 | 영상/다이어리 기반 기록 UX |
| **기록과 커뮤니티 분리** | 오늘 산책 기록은 앱에, 커뮤니티 공유는 카페에 따로 | 기록 → 커뮤니티 공유가 한 흐름 안에 |
| **지역성 약함** | 전국 단위 카페에서 내 동네 정보 찾기 어려움 | 동네 기반 커뮤니티 레이어 |
| **신뢰 기반 관계 없음** | 닉네임 기반, 실제로 산책한 사람인지 알 수 없음 | 실제 산책 기록이 있는 사용자 기반 관계 |

### 핵심 차별화 포인트 (가설)

> **"내 동물을 이미 아는 커뮤니티"**

기존 커뮤니티는 내가 글을 쓸 때마다 내 동물을 소개해야 한다.
이 서비스는 프로필, 산책 기록, 매칭 이력이 이미 쌓여있는 상태에서 커뮤니티로 연결된다.
커뮤니티가 추가 기능이 아니라, 기록과 관계의 자연스러운 다음 단계가 된다.

### 차별화 포인트 2: 반려동물이 주인공인 공간

#### 인스타그램·유튜브와의 근본적인 차이

인스타그램에 반려동물 콘텐츠를 올릴 때의 구조:

```
인스타:  주인(계정 주체) → 반려동물 콘텐츠 업로드 → 주인의 팔로워
이 앱:   반려동물(공간 주체) → 주인이 기록을 채워줌 → 다른 반려동물·반려인
```

인스타그램은 주인이 주인공이고 반려동물은 소재다.
이 앱에서는 반려동물이 자신의 공간·프로필·피드를 가진 주인공이 된다.
유튜브·인스타로 대체될 수 없는 이유가 여기에 있다.

#### 반려동물 의인화 UX

프로필과 공간을 반려동물의 1인칭 관점으로 구성한다.

| 일반적인 반려동물 앱 | 의인화 UX |
|---|---|
| "보미, 말티즈, 3살" | "안녕, 나는 보미야. 느린 산책이 제일 좋아." |
| 주인이 정보를 입력하는 폼 | 반려동물의 목소리로 채워지는 공간 |
| 기록 = 주인의 일기 | 기록 = 보미의 오늘 하루 |

의인화는 기능이 아니라 서비스 전반의 톤앤매너로 자연스럽게 스며들어야 한다.
억지스러운 의인화는 오히려 유치하게 느껴질 수 있어 강도 조절이 중요하다.

#### 반려동물 전용 공간 (Pet Space)

각 반려동물이 자신의 공간(홈)을 갖는다.

- **피드**: 산책 기록, 매칭 기록이 반려동물 관점에서 쌓임
- **공간 꾸미기**: 주인이 반려동물의 방/집을 꾸며줌 (배경, 소품, 스티커 등)
- **방문**: 매칭으로 친해진 다른 반려동물의 공간을 방문할 수 있음

```
실세계 레이어:  산책 매칭 → 실제로 만남
디지털 레이어:  Pet Space → 친구 집에 디지털로 놀러감
```

산책에서 맺어진 관계가 디지털 공간에서도 이어지는 구조.
싸이월드 미니홈피를 반려동물이 소유하는 감각에 가깝다.

#### 리스크

- 반려동물을 의인화해서 바라보는 성향의 유저에게 강하게 작동하지만, 그렇지 않은 유저에겐 거부감이 생길 수 있음
- 의인화 톤이 전체 UX에서 일관되게 유지되어야 함 — 일부만 의인화되면 어색해짐
- Pet Space 꾸미기가 산책 매칭이라는 메인 기능을 희석시키지 않도록 레이어 분리 필요
