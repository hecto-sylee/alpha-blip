# 고객 요청서

## 1. 프로젝트 개요

- 프로젝트명: 스냅챌
- 서비스 형태: Android 기반 퀘스트형 짧은 영상 소셜 앱 MVP
- 한 줄 설명: 친구나 커플이 하루의 퀘스트 주제에 맞춰 매시간 2초 영상을 찍고, 하루 끝에 셋로그처럼 모아보는 폐쇄형 소셜 앱
- 배경: 기존 셋로그는 친한 사람끼리 짧은 영상을 공유하고 하루의 기록을 만드는 장점이 있지만, 시간이 지나면 찍을 것이 없고 놀이 요소가 부족해 친구들이 이탈할 수 있음
- 해결하려는 문제: 사용자가 “무엇을 찍을지” 고민하지 않도록 퀘스트 주제를 제공하고, 친구/커플이 더 풍부한 추억을 만들고 SNS에 공유하고 싶어지는 결과물을 만들도록 함

## 2. 프로젝트 목표

- 핵심 목표: 퀘스트 기반 소셜 앱으로 독립 서비스화
- 사업 목표: 향후 수익화 가능성을 가진 MVP 검증
- 사용자 목표: 하루 끝에 함께 만든 영상을 보며 재미와 추억을 느끼고, 저장/공유하고 싶게 만드는 것
- 성공 기준: 저장/공유 증가, 퀘스트가 재미있다고 느끼는지, 하루 결과물에 대한 만족도
- 실패/주의 기준: 기존 셋로그와 차이가 약함, 퀘스트가 숙제처럼 느껴짐, 촬영이 귀찮아짐, 친구가 참여하지 않아 재미가 떨어짐

## 3. 대상 사용자

- 주요 사용자: 친구 그룹, 커플
- 보조 사용자: 미정
- 사용자 상황: 최소 2명, 권장 3명, 최대 5명이 함께 사용하는 가까운 관계
- 사용자 문제: 찍을 게 없음, 친구가 안 찍음, 놀이 요소 부족, 촬영이 귀찮음
- 기존 대안: 셋로그, 인스타 친친

## 4. 핵심 시나리오

- 첫 사용 시나리오: 방 만들기 -> 방 이름/친구·커플 모드/프로필 이름 입력 -> 초대 링크 또는 참여 코드 공유 -> 참여자는 프로필 이름 입력 후 입장
- 핵심 기능 사용 시나리오: 하루 첫 접근자가 추천 퀘스트 후보 3개 중 하나를 선택 -> 매시간 촬영 직전 해당 시간의 세부 미션 확인 -> 2초 영상 촬영 -> 시간대별 로그에 반영
- 반복 사용 시나리오: 오늘의 퀘스트 확인 -> 시간대별 참여 -> 친구 영상 보기 -> 이모지 반응 -> 하루 결과물 저장/공유
- 최종 결과: 셋로그와 유사한 형태의 하루 영상 로그

## 5. 기능 요구사항

### 필수 기능

- 방 만들기
- 초대 링크 공유
- 참여 코드 입력
- 친구/커플 모드 선택
- 프로필 이름 입력
- 하루 첫 접근자의 퀘스트 선택
- 추천 퀘스트 후보 3개 표시
- 촬영 직전 세부 미션 표시
- 2초 영상 촬영
- 시간대별 로그 보기
- 영상 보기
- 이모지 반응
- 저장/공유
- 캘린더 기반 일자별 로그 보기
- 설정 페이지
- 설정 페이지 내 숨겨진 퀘스트 관리 기능
- 친구 모드 사전 생성 퀘스트 6개
- 커플 모드 사전 생성 퀘스트 6개

### 선택 기능

- 알림
- 워터마크
- 회원가입
- 식물/동물 성장 콘텐츠
- LLM 실시간 퀘스트 생성
- 초대 URL 재발급 정책 고도화

### 제외 기능

- 포인트
- 결제
- DM
- 프로필 꾸미기
- 고객 문의
- 공지/이벤트/푸시 발송
- 신고/차단/내보내기
- 운영자 수동 데이터 수정/삭제 요청 처리

## 6. 화면 요구사항

- 필요한 화면: 방 생성 화면, 방 참여 화면, 시간대별 로그 화면, 촬영 직전 퀘스트 안내 화면, 첫 접근자 퀘스트 선택 화면, 카메라 화면, 영상 보기 화면, 저장/공유 화면, 캘린더 화면, 설정 화면
- 주요 화면별 목적: 방 생성/참여, 오늘의 퀘스트 선택, 세부 미션 확인, 2초 촬영, 친구 영상 확인, 이모지 반응, 저장/공유, 일자별 로그 확인, 사용자 관련 설정 관리
- 화면 이동 흐름: 셋로그 구조를 최대한 참고함
- 촬영 직전 퀘스트 안내: 해당 사용자에게만 표시되며, 사용자가 터치해야 촬영으로 넘어감
- 첫 접근자 퀘스트 선택 화면: 추천 주제명과 다른 주제 선택 버튼을 표시함

## 7. 정책 요구사항

- 회원 정책: MVP에서는 회원가입 없음. 기기값 기반으로 식별
- 결제/환불 정책: MVP 제외
- 콘텐츠 정책: 같은 시간대 영상은 1회만 업로드 가능. 삭제한 경우 같은 시간대 안에서만 재촬영 가능
- 알림 정책: MVP 제외. 고도화 시 셋로그와 유사한 매시간 알림 고려
- 운영 정책: 방장은 없음. 매일 첫 접근자가 해당 일자의 퀘스트를 선택. 한 번 정해진 퀘스트는 변경 불가
- 계정 삭제 정책: 회원가입이 없으므로 MVP에서는 앱 삭제 시 기존 방/영상 접근 복구 없음. 회원가입은 고도화 전략
- 방 나가기 정책: 방 나가기 가능. 사용자가 나가면 해당 사용자의 공유 로그는 삭제/숨김 처리하고, 방 안에서 그 사람 칸 자체를 제거
- 방 삭제 정책: 모든 인원이 방을 나가면 방 삭제

## 8. 데이터 요구사항

- 사용자 입력 데이터: 프로필 이름, 방 이름, 친구/커플 모드, 참여 코드
- 관리자 입력 데이터: 별도 운영자 입력이 아니라 설정 페이지 내 숨겨진 퀘스트 관리 기능에서 MVP용 퀘스트 데이터 관리
- 저장 데이터: 방 이름, 모드, 참여 코드, 초대 링크, 참여자 목록, 생성일, 날짜별 로그, 촬영자, 날짜, 시간대, 주제, 세부 미션, 영상 파일, 반응
- 외부 연동 데이터: MVP에서는 실제 LLM API 연동 없이 사전 생성된 퀘스트 응답 사용
- 분석 이벤트: 방 생성 수, 초대 링크 클릭 수, 참여 전환율, 퀘스트 후보 노출/선택률, 시간대별 촬영률, 미참여율, 저장률, 공유율, 이모지 반응 수, 재방문율, 방 유지율, 모드별 사용률 등 최대한 다양하게 수집
- 개인정보/민감정보: 영상 내용 자체는 AI 분석하지 않음

## 9. 예외 상황 요구사항

- 입력 오류: 잘못된 참여 코드/링크는 “방을 찾을 수 없어요” 표시 후 다시 입력
- 빈 데이터: 미참여 시간대는 셋로그처럼 검은 화면 또는 빈 상태로 표시
- 네트워크 오류: 촬영 중 문제가 생기면 촬영부터 다시 시작. 업로드 실패 시 팝업 표시 및 다시 업로드 버튼 제공
- 권한 거부: 미정
- 인증 실패: 회원가입 없음
- 결제 실패: MVP 제외
- 악용/중복 요청: MVP에서는 깊게 고려하지 않음. 방 나가기는 가능
- 방 정원 초과: 최대 5명이 찬 방은 “다 찬 방이에요” 표시 후 다시 입력
- 퀘스트 생성 실패: 기본 퀘스트 목록에서 대체

## 10. 운영 요구사항

- 관리자 기능: 별도 운영자 관리자 페이지는 MVP에서 제공하지 않음
- 설정 페이지: 사용자가 자신과 관련된 항목을 관리
- 퀘스트 관리: MVP 편의를 위해 설정 페이지 안에 숨겨진 형태로 제공
- 고객 문의: MVP 제외
- 공지/푸시: MVP 제외
- 신고/검수: MVP 제외
- 데이터 관리: 사용자가 영상을 삭제하면 원본 완전 삭제가 아니라 공유되는 로그에서만 삭제/숨김 처리. 모든 인원이 방을 나가면 방 삭제

## 11. 디자인/브랜드 요구사항

- 원하는 분위기: 셋로그처럼 단순하고 가벼움 + 동물의 숲 같은 따뜻하고 아기자기한 느낌
- 참고 서비스: 셋로그
- 피하고 싶은 스타일: 너무 유치함, 감성 과잉, 복잡함
- 브랜드 자산: 서비스명 스냅챌. 로고/브랜드 컬러는 미정
- 접근성/다국어: 한국어만 지원. 쉬운 문구 중심
- 모드별 디자인: 친구/커플 모드는 디자인상 동일하게 유지하고 퀘스트 종류만 다르게 운영

## 12. 기술/출시 요구사항

- 플랫폼: Android only
- 서버 필요 여부: 필요. MVP 시연은 로컬 컴퓨터 서버 사용 가능
- 외부 연동: MVP에서는 LLM API 직접 호출 대신 사전 생성 퀘스트 데이터 사용
- 출시 희망 일정: 2주 내 프로토타입
- 예산 범위: 미정
- 유지보수: 일단 MVP만 고려
- 시연 방식: 사용자 휴대폰에 APK 설치 후 확인
- 시연 대상: 알파팀 공유용
- MVP 테스트 인원: 3명

## 13. MVP 범위

- 첫 버전 필수 범위: 방 만들기 -> 링크/코드로 참여 -> 오늘 퀘스트 선택 -> 2초 촬영 -> 시간대별 로그 보기 -> 저장/공유
- 핵심 검증 가설: 퀘스트는 재미있다
- 후순위 기능: 과거 로그 안정화, 알림, 회원가입, 성장 콘텐츠, LLM 실시간 생성, 워터마크, 수익화
- 제외 가능한 범위: 고도화 운영 기능, 고객 문의, 신고/차단, 결제, 포인트, DM
- 반드시 보여줘야 하는 차별점: 첫 접근자가 퀘스트 주제를 고름, 친구/커플 모드별 퀘스트가 다름, 하루 퀘스트 결과물이 셋로그처럼 모임, SNS 공유가 쉬움
- 절대 포기하면 안 되는 것: 핵심 플로우
- 2주 내 어려우면 포기 가능한 것: 과거 로그 안정화

## 14. 미정 사항

- 권한 거부 시 처리 방식
- 로고/브랜드 컬러
- 초대 URL이 언제/어떻게 바뀌는지
- 사전 생성 퀘스트 데이터 포맷
- 로컬 서버 접속 방식: 같은 Wi-Fi 또는 임시 터널링
- 설정 페이지 안에 숨겨둘 퀘스트 관리 기능의 정확한 노출 방식
- 캘린더 기반 과거 로그의 MVP 안정화 범위
