# 데이터 구조 초안

## 1. 문서 목적

이 문서는 스냅챌 MVP 구현을 위해 필요한 데이터 구조를 초안 수준으로 정리한 문서다.

최종 DB 설계가 아니라, 방/참여자/퀘스트/로그/영상/반응/분석 이벤트를 어떤 단위로 저장해야 하는지 개발팀이 이해할 수 있도록 정리한다.

## 2. 설계 전제

- 플랫폼은 Android only다.
- MVP는 APK 직접 설치로 시연한다.
- 서버는 시연용 로컬 서버를 사용할 수 있다.
- 회원가입/로그인은 없다.
- 사용자는 기기값 기반으로 식별한다.
- 방은 최소 2명, 권장 3명, 최대 5명이다.
- 하루 첫 접근자가 오늘의 퀘스트를 선택한다.
- 오늘의 퀘스트는 하루 동안 변경할 수 없다.
- 영상은 2초다.
- 같은 시간대에는 사용자당 1개의 영상만 공유 로그에 올라간다.
- 영상 내용 자체는 AI가 분석하지 않는다.
- MVP에서는 실제 LLM API 호출 없이 사전 생성 퀘스트 데이터를 사용한다.

## 3. 주요 엔티티 요약

| 엔티티 | 설명 |
| --- | --- |
| Room | 사용자가 함께 로그를 찍는 방 |
| Participant | 방에 참여한 사용자 |
| DeviceIdentity | 회원가입 없는 환경에서 사용자를 구분하기 위한 기기 식별 정보 |
| DailyLog | 특정 방의 특정 날짜 로그 |
| QuestTemplate | 사전 생성된 퀘스트 템플릿 |
| DailyQuest | 특정 날짜에 선택된 오늘의 퀘스트 |
| QuestMission | 오늘의 퀘스트에 포함된 시간대별 세부 미션 |
| VideoClip | 사용자가 특정 시간대에 올린 2초 영상 |
| Reaction | 영상에 남긴 이모지 반응 |
| ShareRecord | 저장/공유 행동 기록 |
| AnalyticsEvent | 분석 이벤트 |

## 4. 엔티티 관계 요약

```
Room 1 ─ N Participant
Room 1 ─ N DailyLog
DailyLog 1 ─ 1 DailyQuest
DailyQuest 1 ─ N QuestMission
DailyLog 1 ─ N VideoClip
Participant 1 ─ N VideoClip
VideoClip 1 ─ N Reaction
Participant 1 ─ N Reaction
Room 1 ─ N AnalyticsEvent
```

## 5. Room

### 설명

친구/커플이 함께 사용하는 하나의 방이다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 방 고유 ID |
| name | string | 방 이름 |
| mode | enum | friend 또는 couple |
| joinCode | string | 참여 코드 |
| inviteUrl | string | 초대 링크 |
| participantIds | string[] | 참여자 ID 목록 |
| maxParticipants | number | 최대 인원. MVP 기준 5 |
| createdAt | datetime | 생성 시각 |
| deletedAt | datetime/null | 삭제 시각 |
| status | enum | active, deleted |

### 정책

- 방장은 없다.
- 참여 코드는 바뀌지 않는다.
- 참여 URL은 바뀔 수 있으나 MVP에서는 변경 조건 미정이다.
- 모든 인원이 나가면 방 삭제 처리한다.

## 6. Participant

### 설명

방에 참여한 사용자다. 회원 계정이 아니라 방 안에서의 참여자 개념이다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 참여자 고유 ID |
| roomId | string | 참여 중인 방 ID |
| deviceId | string | 기기 식별값 |
| displayName | string | 프로필 이름 |
| joinedAt | datetime | 참여 시각 |
| leftAt | datetime/null | 방 나간 시각 |
| status | enum | active, left |

### 정책

- 프로필 이름 길이 제한은 MVP에서 두지 않는다.
- 사용자가 방을 나가면 status를 left로 변경한다.
- 방 안에서는 나간 사용자의 칸 자체를 제거한다.
- 나간 사용자의 공유 로그는 삭제/숨김 처리한다.

## 7. DeviceIdentity

### 설명

회원가입이 없는 MVP에서 같은 기기를 식별하기 위한 정보다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| deviceId | string | 앱에서 생성/보관하는 기기 식별값 |
| createdAt | datetime | 최초 생성 시각 |
| lastSeenAt | datetime | 마지막 접근 시각 |

### 정책

- 앱 삭제 또는 기기 변경 시 기존 방/영상 접근 복구는 MVP에서 제공하지 않는다.
- 회원가입은 고도화 전략이다.

## 8. DailyLog

### 설명

특정 방의 특정 날짜 로그 묶음이다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 일자별 로그 ID |
| roomId | string | 방 ID |
| date | date | 로그 날짜 |
| dailyQuestId | string/null | 오늘의 퀘스트 ID |
| createdAt | datetime | 생성 시각 |
| status | enum | open, closed |

### 정책

- 캘린더 화면에서 날짜별 DailyLog를 조회한다.
- 과거 로그 안정화는 2주 내 어려우면 포기 가능 범위다.

## 9. QuestTemplate

### 설명

사전 생성된 퀘스트 데이터다. MVP에서는 실제 LLM 호출 대신 이 데이터를 사용한다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 퀘스트 템플릿 ID |
| mode | enum | friend 또는 couple |
| title | string | 퀘스트 주제명 |
| description | string | 퀘스트 설명 |
| missionSet | array | 시간대별 세부 미션 목록 |
| isFallback | boolean | 기본 대체 퀘스트 여부 |
| enabled | boolean | 사용 여부 |
| createdAt | datetime | 생성 시각 |
| updatedAt | datetime | 수정 시각 |

### 정책

- 친구 모드 6개, 커플 모드 6개를 준비한다.
- 사용자가 직접 퀘스트를 생성하지 않는다.
- 설정 페이지 안 숨겨진 퀘스트 관리 기능에서 MVP용으로 관리한다.

## 10. DailyQuest

### 설명

특정 날짜에 실제로 선택된 오늘의 퀘스트다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 오늘의 퀘스트 ID |
| roomId | string | 방 ID |
| dailyLogId | string | 일자별 로그 ID |
| date | date | 적용 날짜 |
| templateId | string | 선택된 QuestTemplate ID |
| title | string | 오늘의 퀘스트 주제명 |
| description | string | 오늘의 퀘스트 설명 |
| selectedByParticipantId | string | 하루 첫 접근자 ID |
| selectedAt | datetime | 선택 시각 |
| source | enum | pre_generated, fallback, future_llm |
| locked | boolean | 변경 불가 여부 |

### 정책

- 하루 첫 접근자가 선택한다.
- 한 번 정해지면 변경할 수 없다.
- 모든 참여자에게 같은 DailyQuest가 적용된다.

## 11. QuestMission

### 설명

DailyQuest에 포함된 시간대별 세부 미션이다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 세부 미션 ID |
| dailyQuestId | string | 오늘의 퀘스트 ID |
| hourSlot | string/number | 시간대 |
| title | string | 세부 미션 제목 |
| instruction | string | 촬영 직전 보여줄 안내 문구 |
| order | number | 표시 순서 |

### 정책

- 촬영 직전에 해당 사용자에게만 보여준다.
- 사용자가 안내를 터치해야 촬영 화면으로 넘어간다.
- MVP에서는 하루 퀘스트 선택 시 이미 준비된 missionSet을 사용한다.

## 12. VideoClip

### 설명

사용자가 특정 날짜/시간대에 올린 2초 영상이다.

### 필드 초안

| 필드 | 타입 예시 | 설명 |
| --- | --- | --- |
| id | string | 영상 ID |
| roomId | string | 방 ID |
| dailyLogId | string | 일자별 로그 ID |
| participantId | string | 촬영자 ID |
| dailyQuestId | string | 오늘의 퀘스트 ID |
| questMissionId | string | 해당 시간대 세부 미션 ID |
| date | date | 촬영 날짜 |
| hourSlot | string/number | 시간대 |
| filePath | string | 서버 또는 로컬 저장 경로 |
| durationSec | number | 영상 길이. MVP 기준 2 |
| uploadStatus | enum | uploading, uploaded, failed |
| visibleInSharedLog | boolean | 공유 로그 표시 여부 |
| createdAt | datetime | 촬영/생성 시각 |
| uploadedAt | datetime/null | 업로드 완료 시각 |
| hiddenAt | datetime/null | 공유 로그에서 숨김 처리된 시각 |

### 정책

- 같은 participantId + dailyLogId + hourSlot 조합에는 표시 가능한 영상이 1개만 있어야 한다.
- 사용자가 삭제하면 원본 완전 삭제가 아니라 공유 로그에서 visibleInSharedLog=false 처리한다.
- 삭제한 경우 같은 시간대 안에서 재촬영 가능하다.
- 영상 내용 자체는 AI 분석하지 않는다.

### 필드 초안

## 16. 상태값 초안

### Room.status

- active: 사용 중
- deleted: 삭제됨

### Participant.status

- active: 참여 중
- left: 나감

### DailyLog.status

- open: 진행 중
- closed: 종료됨

### VideoClip.uploadStatus

- uploading: 업로드 중
- uploaded: 업로드 완료
- failed: 업로드 실패

### DailyQuest.source

- pre_generated: 사전 생성 데이터
- fallback: 기본 대체 퀘스트
- future_llm: 향후 LLM 생성

## 17. 주요 제약 조건

### 방 인원 제한

```
Room.participantIds 중 active 상태 참여자는 최대 5명
```

### 시간대별 영상 제한

```
participantId + dailyLogId + hourSlot 기준 visibleInSharedLog=true 영상은 최대 1개
```

### 퀘스트 변경 제한

```
DailyQuest.locked = true 이면 변경 불가
```

### 방 삭제 조건

```
Room에 active 참여자가 0명이 되면 Room.status = deleted
```

## 18. MVP용 저장 방식 참고

MVP에서는 정식 클라우드 서버가 아니라 로컬 서버 시연이 가능하다.

가능한 단순 구성 예시:

- 메타데이터: SQLite, JSON 파일, 또는 간단한 로컬 DB
- 영상 파일: 로컬 서버의 파일 스토리지
- Android 앱: 로컬 서버 API 호출
- 접속 조건: 같은 Wi-Fi 또는 임시 터널링

정식 출시/외부 테스트 단계에서는 클라우드 서버와 안정적인 파일 저장소가 필요하다.

## 19. AI/LLM 관련 데이터 정책

### MVP

- 실제 LLM API 호출 없음
- 사전 생성된 퀘스트 응답 데이터 사용
- 영상 내용 분석 없음
- 참고 데이터는 퀘스트 템플릿과 텍스트 기반 정보로 제한

### 고도화

- 관계 모드
- 오늘의 주제
- 이전 시간대 세부 미션
- 금일 진행한 이전 모든 시간대별 진행 퀘스트 내용

위 정보를 참고해 다음 미션을 동적으로 추천할 수 있음.

## 20. 미정 사항

- 사전 생성 퀘스트 데이터의 최종 JSON 포맷
- hourSlot을 실제 시간으로 둘지, 순번으로 둘지
- 파일 저장 경로/스토리지 방식
- 초대 URL 변경 조건
- 기기값 생성 방식
- 권한 거부 상태 저장 여부
- 저장/공유 결과물 파일 구조
- 과거 로그를 얼마나 안정적으로 보존할지
