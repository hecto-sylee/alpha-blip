# MVP 범위 정의서

## 1. MVP 목적

스냅챌 MVP의 목적은 2주 안에 Android APK로 시연 가능한 프로토타입을 만들고, “퀘스트가 있으면 셋로그형 기록 경험이 더 재미있어지는가”를 알파팀 내부에서 확인하는 것이다.

MVP는 정식 출시용이 아니라 알파팀 공유용 시연 버전이다. 완성도 높은 운영 기능보다 핵심 사용 흐름과 차별점 검증을 우선한다.

## 2. 핵심 검증 가설

- 퀘스트는 재미있다.
- 사용자는 단순히 매시간 영상을 찍는 것보다, 하루의 퀘스트 주제가 있을 때 더 찍을 이유를 느낀다.
- 친구/커플 관계에 맞는 퀘스트가 있으면 결과물이 더 추억처럼 느껴진다.
- 하루 끝에 만들어진 로그는 저장/공유하고 싶어질 수 있다.

## 3. MVP 성공 기준

- 3명이 같은 방에 참여할 수 있다.
- 하루 첫 접근자가 퀘스트 후보 3개 중 하나를 선택할 수 있다.
- 선택된 퀘스트 주제에 따라 시간대별 세부 미션이 표시된다.
- 사용자가 2초 영상을 촬영하고 시간대별 로그에서 볼 수 있다.
- 미참여 시간대는 셋로그처럼 빈 화면 또는 검은 화면으로 표시된다.
- 이모지 반응이 가능하다.
- 하루 로그를 저장/공유할 수 있다.
- 알파팀에게 “퀘스트가 있는 셋로그형 앱”이라는 차별점이 전달된다.

## 4. MVP 핵심 플로우

1. 사용자가 방을 만든다.
2. 방 이름, 친구/커플 모드, 프로필 이름을 입력한다.
3. 초대 링크 또는 참여 코드를 공유한다.
4. 다른 사용자가 초대 링크 또는 참여 코드로 입장한다.
5. 참여자는 프로필 이름을 입력하고 바로 참여한다.
6. 하루 첫 접근자가 추천 퀘스트 후보 3개 중 하나를 선택한다.
7. 선택된 퀘스트가 그날 하루의 로그 주제가 된다.
8. 사용자는 시간대별로 촬영 직전 세부 미션을 확인한다.
9. 사용자가 안내를 터치하면 촬영 화면으로 이동한다.
10. 사용자는 2초 영상을 촬영한다.
11. 촬영된 영상은 해당 시간대 로그에 표시된다.
12. 사용자는 친구/커플의 영상을 보고 이모지 반응을 남긴다.
13. 하루 로그를 저장/공유한다.

## 5. MVP 필수 범위

### 방/참여

- 방 만들기
- 방 이름 입력
- 친구/커플 모드 선택
- 프로필 이름 입력
- 초대 링크 생성
- 참여 코드 생성
- 초대 링크로 참여
- 참여 코드 입력으로 참여
- 방 최대 인원 5명 제한
- MVP 실제 테스트 인원 3명 기준 동작

### 퀘스트

- 하루 첫 접근자 판별
- 추천 퀘스트 후보 3개 표시
- 첫 접근자가 후보 중 1개 선택
- 선택된 퀘스트는 하루 동안 변경 불가
- 친구 모드 퀘스트 6개 준비
- 커플 모드 퀘스트 6개 준비
- 선택된 주제에 따른 시간대별 세부 미션 표시
- 퀘스트 생성 실패 시 기본 퀘스트 목록에서 대체

### 촬영/로그

- 촬영 직전 퀘스트 안내 표시
- 퀘스트 안내 터치 후 촬영 화면 이동
- 2초 영상 촬영
- 같은 시간대 1회 업로드 제한
- 삭제한 경우 같은 시간대 안에서만 재촬영 가능
- 시간대별 로그 보기
- 미참여 시간대 빈 화면 또는 검은 화면 표시
- 캘린더 기반 일자별 로그 보기

### 반응/공유

- 영상 보기
- 이모지 반응
- 하루 로그 저장
- 하루 로그 공유

### 설정

- 설정 페이지 제공
- 사용자가 자신과 관련된 항목만 관리
- 방 나가기
- MVP 편의를 위한 숨겨진 퀘스트 관리 기능

### 서버/데이터

- 로컬 서버 기반 시연 가능 구조
- 방 데이터 저장
- 참여자 데이터 저장
- 영상 데이터 저장
- 퀘스트 데이터 저장
- 반응 데이터 저장
- 분석 이벤트 수집 구조 준비

## 6. MVP 제외 범위

- iOS 앱
- 정식 앱스토어/플레이스토어 출시
- 회원가입/로그인
- 계정 복구
- 기기 변경 대응
- 알림
- LLM API 실시간 호출
- 영상 내용 AI 분석
- 포인트
- 결제
- DM
- 프로필 꾸미기
- 고객 문의
- 공지/이벤트
- 푸시 발송
- 신고
- 차단
- 내보내기
- 운영자용 별도 관리자 페이지
- 운영자 수동 데이터 수정/삭제 요청 처리
- 워터마크
- 식물/동물 성장 콘텐츠
- 수익화 기능

## 7. 후순위 / 고도화 범위

- 알림: 셋로그와 유사한 매시간 알림
- 회원가입: 기기 변경, 계정 복구, 장기 사용 대응
- LLM 실시간 생성: 관계 모드, 오늘의 주제, 이전 시간대 세부 미션을 참고한 동적 추천
- 식물/동물 성장 콘텐츠: 출석체크, 공유, 퀘스트 성공률에 따라 함께 키우는 물체 성장
- 워터마크: SNS 공유 시 브랜드 노출
- 수익화: 추후 별도 검토
- 초대 URL 재발급 정책
- 과거 로그 안정화
- 신고/차단/내보내기 등 안전 기능
- 운영자용 관리자 페이지

## 8. 2주 내 포기 가능 범위

2주 안에 구현이 어렵다면 아래 항목은 우선순위를 낮춘다.

- 과거 로그 안정화
- 캘린더의 완성도 높은 탐색 경험
- 분석 이벤트 상세 대시보드
- 숨겨진 퀘스트 관리 기능의 사용성
- 저장/공유 결과물의 고급 편집

단, 핵심 플로우는 포기하지 않는다.

## 9. 절대 포기하면 안 되는 범위

- 방 만들기
- 링크/코드로 참여
- 친구/커플 모드 선택
- 하루 첫 접근자의 퀘스트 선택
- 퀘스트 후보 3개 표시
- 촬영 직전 세부 미션 표시
- 2초 촬영
- 시간대별 로그 보기
- 저장/공유

## 10. 주요 정책

- 회원가입은 없다.
- 사용자는 기기값 기반으로 식별한다.
- 방장은 없다.
- 매일 하루의 첫 번째 접근자가 퀘스트를 선택한다.
- 한 번 정해진 퀘스트는 변경할 수 없다.
- 같은 시간대에는 1회만 촬영 가능하다.
- 삭제한 경우 같은 시간대 안에서만 다시 촬영할 수 있다.
- 못 찍은 시간대는 실패/미참여로 처리한다.
- 건너뛰기 버튼은 없다.
- 미참여 시간대는 셋로그처럼 빈 화면 또는 검은 화면으로 표시한다.
- 방 나가기는 가능하다.
- 사용자가 나가면 해당 사용자의 공유 로그는 삭제/숨김 처리하고 칸 자체를 제거한다.
- 모든 인원이 나가면 방을 삭제한다.

## 11. 예외 처리 범위

- 잘못된 참여 코드 또는 링크: “방을 찾을 수 없어요” 표시 후 다시 입력
- 방 정원 초과: “다 찬 방이에요” 표시 후 다시 입력
- 촬영 중 오류: 촬영부터 다시 시작
- 업로드 실패: 팝업 표시 후 다시 업로드 버튼 제공
- 퀘스트 생성 실패: 기본 퀘스트 목록에서 대체
- 권한 거부: 미정

## 12. 기술 범위

- 플랫폼: Android only
- 배포: APK 직접 설치
- 서버: MVP 시연용 로컬 서버 가능
- 로컬 서버 접속 조건: 같은 Wi-Fi 또는 임시 터널링 필요
- LLM: MVP에서는 실제 API 호출 없음
- 퀘스트 데이터: 사전 생성된 응답 데이터 사용
- 영상 분석: 하지 않음

## 13. 시연 범위

- 시연 대상: 알파팀
- 시연 인원: 3명
- 시연 목표: 스냅챌의 핵심 차별점인 “하루 퀘스트가 있는 셋로그형 경험” 전달
- 시연 성공 장면: 방 생성 -> 3명 참여 -> 첫 접근자 퀘스트 선택 -> 시간대별 2초 촬영 -> 로그 보기 -> 저장/공유

## 14. 남은 결정사항

- 권한 거부 시 사용자 안내 방식
- 사전 생성 퀘스트 데이터 포맷
- 로컬 서버 접속 방식 확정
- 초대 URL 변경 조건
- 설정 페이지 안의 숨겨진 퀘스트 관리 기능 노출 방식
- 캘린더 기반 과거 로그를 MVP에서 어디까지 안정화할지
