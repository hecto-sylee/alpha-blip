# 퀘스트 콘텐츠 설계서

## 1. 문서 목적

이 문서는 스냅챌 MVP에서 사용할 퀘스트 콘텐츠의 구조, 작성 규칙, 데이터 포맷, 초기 퀘스트 주제를 정의한다.

스냅챌의 퀘스트는 단발 미션이 아니라, 하루 동안 어떤 로그를 만들지 정하는 “하루의 로그 주제”다.

## 2. 참고 문서

- 퀘스트 종류 조사: [퀘스트 종류 조사](https://www.notion.so/361cb9b019598041a8a2f1fdcbaedf30?pvs=21)

참고 문서에서 확인한 주요 퀘스트 유형:

- 무지개 색깔 로그
- 같은 포즈 챌린지
- 우리는 하나 컨셉
- 같이 만나서 찍기
- 내 주변 가장 웃긴 거 찍기
- 핀터레스트 로그
- 금지 장면 게임
- 공통 주제로 찍기

## 3. 퀘스트의 역할

퀘스트는 사용자가 “지금 뭘 찍지?”라고 고민하는 시간을 줄이고, 하루의 영상 로그가 하나의 컨셉으로 묶이게 하는 장치다.

기존 셋로그의 문제였던 “찍을 게 없음”, “놀이 요소 부족”, “친구 이탈”을 줄이는 것이 목적이다.

## 4. 퀘스트 기본 정책

- 퀘스트는 하루 단위로 적용된다.
- 하루 첫 접근자가 추천 후보 3개 중 1개를 선택한다.
- 한 번 선택된 퀘스트는 그날 변경할 수 없다.
- 퀘스트는 친구 모드와 커플 모드에 따라 다르게 추천된다.
- MVP에서는 실제 LLM API 호출 없이 사전 생성된 퀘스트 데이터를 사용한다.
- 사용자는 직접 퀘스트를 생성하지 않는다.
- 퀘스트 생성 실패 시 기본 퀘스트 목록에서 대체한다.
- 영상 내용 자체는 AI가 분석하지 않는다.

## 5. 퀘스트 추천 방식

### MVP

- 친구 모드 퀘스트 주제 6개 준비
- 커플 모드 퀘스트 주제 6개 준비
- 각 주제는 06:00부터 23:00까지 18개 시간대별 미션을 가진다.
- 하루 첫 접근자에게 해당 모드의 퀘스트 후보 3개 노출
- 첫 접근자가 1개 선택
- 선택된 퀘스트의 시간대별 미션을 촬영 직전에 표시

### 고도화

향후 LLM을 사용할 경우 아래 정보를 참고할 수 있다.

- 관계 모드: 친구/커플
- 오늘의 주제
- 금일 진행한 이전 모든 시간대별 진행 퀘스트 내용
- 이전 시간대별 미션

단, 영상 내용 자체는 분석하지 않는다.

## 6. 퀘스트 데이터 구조

### 필수 필드

| 필드 | 설명 |
| --- | --- |
| questId | 퀘스트 주제 고유 ID |
| mode | friend 또는 couple |
| title | 퀘스트 주제명 |
| shortDescription | 후보 카드에 보여줄 짧은 설명 |
| fullDescription | 상세 설명 |
| tone | 퀘스트 분위기 |
| difficulty | 쉬움/보통/어려움 |
| missions | 06:00~23:00 시간대별 미션 목록 |
| fallback | 기본 대체 퀘스트 여부 |
| tags | 색상, 포즈, 감성, 웃김, 협동 등 분류 태그 |

### missions 필드 구조

| 필드 | 설명 |
| --- | --- |
| order | 미션 순서 |

| missionTitle | 미션 제목 |
| instruction | 실제 사용자에게 보이는 직관 문구 |
| example | 예시 |

## 7. 퀘스트 작성 원칙

### 좋은 퀘스트 기준

- 한눈에 이해되고 바로 촬영 행동이 떠올라야 한다.
- 사용자 노출 문구는 짧은 명령형 또는 질문형을 우선한다.
- 찍을 대상이 너무 어렵지 않아야 한다.
- 같은 주제로 찍어도 사람마다 결과물이 달라야 한다.
- 하루 끝에 모아봤을 때 하나의 컨셉이 보여야 한다.
- 친구/커플이 서로의 차이를 보는 재미가 있어야 한다.
- 촬영이 부끄럽거나 부담스럽지 않아야 한다.
- 한국어 문구는 짧고 쉬워야 한다.

### 피해야 할 퀘스트

- 너무 유치한 것
- 감성이 과한 것
- 설명이 긴 것
- 특정 장소나 물건이 없으면 거의 불가능한 것
- 개인정보나 민감한 상황을 드러내게 하는 것
- 위험한 행동을 유도하는 것
- 사용자 노출 문구가 장황한 것

## 8. 후보 카드 문구 규칙

첫 접근자에게 보여주는 후보 카드는 짧고 바로 이해되어야 한다.

### 카드 구성

- 주제명
- 한 줄 설명
- 선택 버튼

### 예시

```
무지개 로그
시간마다 다른 색을 찾아요.
```

```
오늘의 작은 웃김
주변에서 제일 웃긴 순간을 찾아요.
```

## 9. 촬영 직전 문구 규칙

촬영 직전 문구는 실제 사용자가 바로 행동할 수 있게 작성한다.

### 권장 형식

```
빨간색 물건을 찾아라!!
```

```
지금 가장 가까운 물건은?
```

```
지금 당신의 기분을 한손으로 표현하라!
```

### 문구 톤

- 짧은 명령형 또는 질문형
- 설명보다 행동 중심
- 한 문장 안에서 바로 찍을 대상이 떠올라야 함
- 20자 안팎 권장
- 사용자 노출 문구에는 영상 길이를 언급하지 않음

| 22:00 | 밤공기 | 밤공기 느낌은? | 창밖, 길 |

## 13. 기본 대체 퀘스트

퀘스트 후보 생성 또는 조회에 실패했을 때 사용할 기본 퀘스트다.

### 기본 후보

1. 공통 주제 로그
2. 무지개 로그
3. 같은 하루 다른 시선

### 정책

- 모드별 후보를 가져오지 못하면 기본 후보에서 대체한다.
- 기본 후보는 촬영 난이도가 낮고, 특정 상황에 덜 의존하는 퀘스트로 구성한다.

## 15. 사전 생성용 프롬프트 초안

MVP에서는 사용자가 사전에 퀘스트 응답을 생성해오는 방식이다. 아래 프롬프트를 사용해 퀘스트 데이터를 만들 수 있다.

```
너는 짧은 영상 소셜 앱 '스냅챌'의 퀘스트 콘텐츠 기획자다.

스냅챌은 친구 또는 커플이 하루 동안 같은 주제에 맞춰 시간대별 영상을 찍고, 하루 끝에 셋로그처럼 모아보는 앱이다.

아래 조건에 맞춰 퀘스트 데이터를 만들어라.

조건:
- 퀘스트는 단발 미션이 아니라 하루의 로그 주제다.
- 각 퀘스트 주제는 시간대별 미션 18개를 가진다.
- 시간대는 06:00부터 23:00까지 빠짐없이 사용한다.
- 실제 사용자에게 보이는 instruction은 짧은 명령형 또는 질문형으로 작성한다.
- instruction에는 영상 길이를 언급하지 않는다.
- instruction에는 장황한 설명을 넣지 않는다.
- 문구는 한국어로 쉽고 직관적으로 작성한다.
- 너무 유치하거나 감성 과잉이면 안 된다.
- 위험한 행동, 개인정보 노출, 민감한 상황을 유도하지 않는다.
- 친구 모드와 커플 모드를 구분한다.
- 출력은 JSON 배열로 한다.

필드:
- questId
- mode: friend 또는 couple
- title
- shortDescription
- fullDescription
- tone
- difficulty: easy, normal, hard 중 하나
- fallback: true 또는 false
- tags
- missions: order, time, missionTitle, instruction, example

생성할 내용:
- friend 모드 퀘스트 주제 6개
- couple 모드 퀘스트 주제 6개
- 각 주제별 missions 18개
```

## 16. 콘텐츠 검수 체크리스트

퀘스트를 추가하거나 수정할 때 아래 기준을 확인한다.

- 사용자가 바로 이해할 수 있는가?
- 사용자 노출 문구가 짧은 명령형 또는 질문형인가?
- 특정 장소/상황이 없어도 할 수 있는가?
- 하루 끝에 모아봤을 때 컨셉이 보이는가?
- 친구/커플 관계에 맞는가?
- 너무 유치하지 않은가?
- 감정 표현을 과하게 요구하지 않는가?
- 개인정보나 민감한 정보가 노출될 위험이 낮은가?
- 위험하거나 무리한 행동을 유도하지 않는가?
- 미참여가 생겨도 로그가 망가지지 않는가?
- 06:00~23:00 시간대가 빠짐없이 들어갔는가?

## 17. 고도화 아이디어

- LLM이 이전 시간대 미션을 참고해 다음 미션 추천
- 방별 분위기에 맞는 퀘스트 추천
- 계절/날씨/기념일 기반 퀘스트
- 공유 결과물에 퀘스트명 또는 워터마크 추가
- 출석체크/공유/퀘스트 성공률 기반 식물·동물 성장 콘텐츠 연동
- 친구/커플 외 관계 모드 추가

## 18. 미정 사항

- 실제 MVP에서 사용할 최종 12개 퀘스트 주제 확정 여부
- 앱 내 숨겨진 퀘스트 관리 화면에서 수정 가능한 필드 범위
- 후보 3개를 랜덤으로 뽑을지, 순환 방식으로 뽑을지
- 퀘스트별 난이도 표시를 사용자에게 보여줄지 여부
- 저장/공유 결과물에 퀘스트명을 노출할지 여부
