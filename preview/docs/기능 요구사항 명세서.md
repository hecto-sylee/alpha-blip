# 기능 요구사항 명세서

## 1. 문서 목적

이 문서는 스냅챌 MVP에서 구현해야 하는 기능 요구사항을 기능 단위로 정리한 문서다.

MVP 목표는 2주 안에 Android APK로 시연 가능한 프로토타입을 만들고, “퀘스트가 있는 셋로그형 경험”을 알파팀에 보여주는 것이다.

## 2. 기능 범위 요약

### MVP 포함

- 방 만들기
- 초대 링크/참여 코드로 방 참여
- 친구/커플 모드 선택
- 프로필 이름 입력
- 하루 첫 접근자의 퀘스트 선택
- 추천 퀘스트 후보 3개 표시
- 촬영 직전 세부 미션 표시
- 2초 영상 촬영
- 시간대별 로그 보기
- 미참여 시간대 빈/검은 화면 표시
- 영상 보기
- 이모지 반응
- 저장/공유
- 캘린더 기반 일자별 로그 보기
- 설정 페이지
- 방 나가기
- 숨겨진 퀘스트 관리 기능

### MVP 제외

- iOS
- 회원가입/로그인
- 알림
- 실제 LLM API 호출
- 영상 내용 AI 분석
- 포인트/결제
- DM
- 프로필 꾸미기
- 고객 문의
- 공지/푸시
- 신고/차단/내보내기
- 별도 운영자 관리자 페이지
- 워터마크
- 식물/동물 성장 콘텐츠

## 3. 공통 정책

- 플랫폼은 Android only다.
- MVP는 APK 직접 설치 방식으로 시연한다.
- 사용자는 회원가입 없이 이용한다.
- 사용자는 기기값 기반으로 식별한다.
- 방은 최소 2명, 권장 3명, 최대 5명 기준이다.
- MVP 실제 테스트 인원은 3명이다.
- 방장은 없다.
- 매일 첫 번째 시간 로그에 가장 먼저 접근한 사용자가 그날의 퀘스트를 선택한다.
- 한 번 정해진 오늘의 퀘스트는 변경할 수 없다.
- 영상은 2초 길이로 촬영한다.
- 같은 시간대에는 사용자당 1회만 업로드할 수 있다.
- 삭제한 경우에만 같은 시간대 안에서 재촬영할 수 있다.
- 미참여 시간대는 실패/미참여로 처리한다.
- 건너뛰기 버튼은 없다.

## 4. 기능 요구사항

## F-001. 방 만들기

### 설명

사용자가 새로운 스냅챌 방을 생성하는 기능이다.

### 입력값

- 방 이름
- 모드
    - 친구 모드
    - 커플 모드
- 프로필 이름

### 동작

1. 사용자가 방 만들기를 선택한다.
2. 사용자가 방 이름을 입력한다.
3. 사용자가 친구/커플 모드를 선택한다.
4. 사용자가 프로필 이름을 입력한다.
5. 앱이 방을 생성한다.
6. 앱이 참여 코드와 초대 링크를 생성한다.
7. 방 생성자는 해당 방의 첫 참여자로 등록된다.

### 정책

- 방 이름과 프로필 이름의 글자 수 제한은 MVP에서는 두지 않는다.
- 방 생성자는 별도 방장이 되지 않는다.
- 방장은 존재하지 않는다.

### 예외

- 방 생성 실패 시 다시 시도하도록 안내한다.

## F-002. 초대 링크 생성/공유

### 설명

방 참여를 위해 초대 링크를 생성하고 공유하는 기능이다.

### 동작

1. 방 생성 후 초대 링크가 생성된다.
2. 사용자는 초대 링크를 외부 앱으로 공유할 수 있다.
3. 초대 링크를 받은 사용자는 링크를 통해 방 참여 화면으로 이동한다.

### 정책

- 참여 URL은 바뀔 수 있다.
- 참여 URL 변경 조건은 MVP에서 미정이다.
- 원치 않는 유입 방지는 MVP에서 깊게 다루지 않는다.

## F-003. 참여 코드 생성/입력

### 설명

초대 링크 대신 참여 코드를 입력해 방에 들어가는 기능이다.

### 동작

1. 방 생성 시 참여 코드가 생성된다.
2. 참여자는 앱에서 참여 코드 입력을 선택한다.
3. 참여자가 코드를 입력한다.
4. 앱이 해당 방을 찾는다.
5. 방이 존재하면 프로필 이름 입력 단계로 이동한다.

### 정책

- 참여 코드는 바뀌지 않는다.

### 예외

- 잘못된 코드: “방을 찾을 수 없어요” 표시 후 다시 입력
- 방이 가득 찬 경우: “다 찬 방이에요” 표시 후 다시 입력

## F-004. 방 참여

### 설명

초대 링크 또는 참여 코드로 방에 들어오는 기능이다.

### 입력값

- 프로필 이름

### 동작

1. 사용자가 초대 링크 또는 참여 코드로 방에 접근한다.
2. 사용자가 프로필 이름을 입력한다.
3. 앱이 사용자를 참여자 목록에 추가한다.
4. 사용자는 방의 로그 화면으로 이동한다.

### 정책

- 회원가입은 없다.
- 기기값 기반으로 사용자를 식별한다.
- 방 최대 인원은 5명이다.
- 프로필 이름 제한은 MVP에서 두지 않는다.

## F-005. 모드 선택

### 설명

방 생성 시 친구 모드 또는 커플 모드를 선택하는 기능이다.

### 동작

1. 방 생성자가 모드를 선택한다.
2. 선택된 모드는 방 정보에 저장된다.
3. 모드에 따라 추천되는 퀘스트 종류가 달라진다.

### 정책

- 친구/커플 모드는 디자인상 동일하게 유지한다.
- 차이는 퀘스트 종류 중심으로 둔다.

## F-006. 오늘의 퀘스트 후보 준비

### 설명

하루 첫 접근자가 선택할 수 있도록 추천 퀘스트 후보 3개를 준비하는 기능이다.

### 동작

1. 새로운 날짜에 아직 오늘의 퀘스트가 없으면 후보 3개를 준비한다.
2. 방의 모드에 맞는 퀘스트 후보를 보여준다.
3. MVP에서는 사전 생성된 퀘스트 데이터에서 후보를 가져온다.

### 정책

- MVP에서는 실제 LLM API를 호출하지 않는다.
- 친구 모드 퀘스트 6개, 커플 모드 퀘스트 6개를 사전 준비한다.
- 퀘스트는 단일 미션이 아니라 하루의 로그 주제다.

### 예외

- 후보 준비 실패 시 기본 퀘스트 목록에서 대체한다.

## F-007. 하루 첫 접근자 판별

### 설명

오늘의 퀘스트를 선택할 사용자를 결정하는 기능이다.

### 동작

1. 사용자가 해당 날짜의 첫 번째 시간 로그에 접근한다.
2. 앱이 오늘의 퀘스트가 이미 정해졌는지 확인한다.
3. 오늘의 퀘스트가 없으면 가장 먼저 접근한 사용자를 하루 첫 접근자로 판단한다.
4. 해당 사용자에게 퀘스트 선택 화면을 보여준다.

### 정책

- 하루 첫 접근자는 방장이 아니다.
- 매일 달라질 수 있다.
- 동시에 여러 명이 접근할 경우 한 명만 선택 권한을 가져야 한다.

## F-008. 오늘의 퀘스트 선택

### 설명

하루 첫 접근자가 추천 후보 3개 중 하나를 선택하는 기능이다.

### 화면 요소

- 추천 주제명
- 후보 3개
- 다른 주제 선택 버튼 또는 후보 선택 영역

### 동작

1. 앱이 후보 3개를 보여준다.
2. 첫 접근자가 후보 중 하나를 선택한다.
3. 선택된 퀘스트가 오늘의 로그 주제로 저장된다.
4. 이후 모든 참여자에게 같은 오늘의 퀘스트가 적용된다.

### 정책

- 한 번 선택된 퀘스트는 변경할 수 없다.
- 참여자가 직접 퀘스트를 새로 생성하지 않는다.

## F-009. 시간대별 세부 미션 표시

### 설명

오늘의 퀘스트 주제에 따라 각 시간대에 수행할 세부 미션을 촬영 직전에 보여주는 기능이다.

### 동작

1. 사용자가 촬영 가능한 시간대에 접근한다.
2. 앱이 해당 시간대의 세부 미션을 확인한다.
3. 촬영 직전 세부 미션을 해당 사용자에게만 보여준다.
4. 사용자가 세부 미션 안내를 터치하면 촬영 화면으로 이동한다.

### 정책

- 세부 미션은 촬영 직전에 보여준다.
- 터치해야 촬영 화면으로 넘어간다.
- 미션 안내는 당사자에게만 보인다.

## F-010. 2초 영상 촬영

### 설명

사용자가 세부 미션에 맞춰 2초 영상을 촬영하는 기능이다.

### 동작

1. 사용자가 촬영 화면에 진입한다.
2. 앱이 카메라를 실행한다.
3. 사용자가 2초 영상을 촬영한다.
4. 촬영된 영상을 업로드한다.
5. 업로드 완료 후 해당 시간대 로그에 반영한다.

### 정책

- 영상 길이는 2초다.
- 같은 시간대에는 사용자당 1회만 촬영/업로드 가능하다.
- 삭제된 경우에만 같은 시간대 안에서 재촬영 가능하다.

### 예외

- 촬영 중 앱 오류: 촬영부터 다시 시작
- 업로드 실패: 팝업 표시 후 다시 업로드 버튼 제공

## F-011. 업로드 재시도

### 설명

영상 업로드 실패 시 사용자가 다시 업로드할 수 있도록 하는 기능이다.

### 동작

1. 업로드 실패 시 팝업을 표시한다.
2. 팝업에서 다시 업로드 버튼을 제공한다.
3. 사용자가 다시 업로드를 누르면 동일 영상을 재업로드한다.

### 정책

- 업로드 실패를 조용히 넘기지 않는다.
- 사용자가 실패 상태를 알 수 있어야 한다.

## F-012. 시간대별 로그 보기

### 설명

시간대별로 참여자들의 영상을 확인하는 기능이다.

### 동작

1. 사용자가 로그 화면에 진입한다.
2. 앱이 날짜와 시간대별 로그를 보여준다.
3. 사용자가 시간대를 선택한다.
4. 해당 시간대의 참여자 영상이 표시된다.
5. 미참여자는 빈 화면 또는 검은 화면으로 표시된다.

### 정책

- 셋로그 구조를 최대한 참고한다.
- 참여자별 칸을 기준으로 보여준다.
- 사용자가 방을 나가면 그 사람 칸 자체를 제거한다.

## F-013. 미참여 표시

### 설명

촬영하지 않은 시간대를 빈 화면 또는 검은 화면으로 표시하는 기능이다.

### 동작

1. 특정 사용자가 해당 시간대에 촬영하지 않는다.
2. 앱은 해당 칸을 빈 화면 또는 검은 화면으로 표시한다.
3. 다음 시간대에는 다시 참여할 수 있다.

### 정책

- 건너뛰기 버튼은 없다.
- 못 찍으면 해당 시간대는 실패/미참여로 처리한다.

## F-014. 영상 보기

### 설명

사용자가 시간대별로 올라온 영상을 재생해 보는 기능이다.

### 동작

1. 사용자가 특정 시간대 또는 특정 참여자 영상을 선택한다.
2. 앱이 영상을 재생한다.
3. 사용자는 다른 참여자의 영상도 볼 수 있다.

### 정책

- 영상 표시/재생 방식은 셋로그를 참고한다.

## F-015. 이모지 반응

### 설명

사용자가 영상에 가볍게 반응할 수 있는 기능이다.

### 동작

1. 사용자가 영상을 본다.
2. 사용자가 이모지 반응을 선택한다.
3. 앱이 해당 영상에 반응을 저장한다.
4. 다른 참여자가 반응을 확인할 수 있다.

### 정책

- MVP에서는 이모지 반응이면 충분하다.
- 댓글, DM은 제공하지 않는다.

## F-016. 영상 삭제

### 설명

사용자가 오늘 자신이 올린 영상을 공유 로그에서 삭제/숨김 처리하는 기능이다.

### 동작

1. 사용자가 자신이 오늘 올린 영상을 선택한다.
2. 사용자가 삭제를 선택한다.
3. 앱이 해당 영상을 공유 로그에서 삭제/숨김 처리한다.
4. 같은 시간대 안이라면 다시 촬영할 수 있다.

### 정책

- 내가 오늘 올린 영상만 삭제할 수 있다.
- 다른 사용자의 영상은 삭제할 수 없다.
- 원본 완전 삭제가 아니라 공유되는 로그에서만 삭제/숨김 처리한다.

## F-017. 저장/공유

### 설명

하루 로그 결과물을 저장하거나 외부 SNS로 공유하는 기능이다.

### 동작

1. 사용자가 하루 로그를 확인한다.
2. 사용자가 저장 또는 공유를 선택한다.
3. 앱이 셋로그와 유사한 형태의 공유 결과물을 제공한다.
4. 사용자가 기기에 저장하거나 외부 앱으로 공유한다.

### 정책

- MVP 성공 기준 중 하나는 저장/공유 증가다.
- 워터마크는 MVP에서는 제외한다.
- 워터마크는 고도화 전략으로 둔다.

## F-018. 캘린더 기반 일자별 로그 보기

### 설명

날짜별로 과거 로그를 확인하는 기능이다.

### 동작

1. 사용자가 캘린더 화면에 진입한다.
2. 사용자가 특정 날짜를 선택한다.
3. 해당 날짜의 로그를 확인한다.

### 정책

- 셋로그에 있는 캘린더 기능을 참고한다.
- 2주 내 구현이 어렵다면 과거 로그 안정화는 포기 가능 범위다.

## F-019. 방 나가기

### 설명

사용자가 참여 중인 방에서 나가는 기능이다.

### 동작

1. 사용자가 설정 페이지에 진입한다.
2. 사용자가 방 나가기를 선택한다.
3. 앱이 사용자를 참여자 목록에서 제거한다.
4. 해당 사용자의 공유 로그는 삭제/숨김 처리한다.
5. 방 안에서 해당 사용자의 칸 자체를 제거한다.
6. 모든 인원이 나간 경우 방을 삭제한다.

### 정책

- 방 나가기는 MVP에 포함한다.
- 모든 인원이 나가면 방 삭제한다.

## F-020. 설정 페이지

### 설명

사용자가 자신과 관련된 항목을 관리하는 화면이다.

### 동작

1. 사용자가 설정 페이지에 진입한다.
2. 사용자가 자신과 관련된 정보를 확인한다.
3. 사용자는 방 나가기를 수행할 수 있다.
4. MVP 편의를 위해 숨겨진 퀘스트 관리 기능에 접근할 수 있다.

### 정책

- 별도 운영자용 관리자 페이지는 만들지 않는다.
- 사용자용 설정 페이지에 가깝다.
- 관리자는 단순히 자신과 관련된 것만 관리한다.

## F-021. 숨겨진 퀘스트 관리 기능

### 설명

MVP 편의를 위해 사전 생성 퀘스트 데이터를 간단히 관리하는 기능이다.

### 동작

1. 설정 페이지 내 숨겨진 진입 방식으로 접근한다.
2. 친구 모드 퀘스트 데이터를 확인/수정할 수 있다.
3. 커플 모드 퀘스트 데이터를 확인/수정할 수 있다.

### 정책

- 정식 운영자 관리자 페이지가 아니다.
- MVP용 임시 기능이다.
- UI 완성도는 낮아도 된다.
- 친구 모드 6개, 커플 모드 6개 퀘스트를 관리한다.

## F-022. 분석 이벤트 수집

### 설명

향후 판단을 위해 사용자 행동 데이터를 수집할 수 있는 구조를 준비한다.

### 수집 후보

- 방 생성 수
- 초대 링크 클릭 수
- 참여 전환율
- 퀘스트 후보 노출 수
- 퀘스트 선택률
- 시간대별 촬영률
- 미참여율
- 저장률
- 공유율
- 이모지 반응 수
- 재방문율
- 방 유지율
- 모드별 사용률

### 정책

- “최대한 다양하게 많이” 수집할 수 있도록 설계한다.
- MVP에서 별도 대시보드는 필수 아님.

## 7. 권한/인증 요구사항

- 회원가입은 없다.
- 로그인은 없다.
- 기기값 기반으로 사용자를 구분한다.
- 앱 삭제 또는 기기 변경 시 기존 방/영상 접근 복구는 MVP에서 제공하지 않는다.
- 카메라/저장소 권한 거부 시 처리 방식은 미정이다.

## 8. 제외 기능 상세

- 알림: MVP 제외. 고도화 시 셋로그와 유사한 매시간 알림 고려
- LLM 실시간 생성: MVP 제외. 사전 생성 응답 사용
- 영상 AI 분석: 개인정보/민감도 때문에 MVP에서는 분석하지 않음
- 회원가입: 고도화 전략
- 성장 콘텐츠: 출석체크, 공유, 퀘스트 성공률 기반 식물/동물 성장 기능은 후순위
- 워터마크: SNS 공유 고도화 시 검토
- 신고/차단/내보내기: MVP 제외
